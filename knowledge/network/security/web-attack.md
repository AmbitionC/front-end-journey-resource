| 攻击类型 | 描述 |
| :---: | --- |
| SQL注入 | 向应用程序的数据库插入恶意SQL代码，以执行恶意操作 |
| 跨站脚本攻击（XSS） | 向网页注入恶意脚本代码，以获取用户信息或执行恶意操作 |
| 跨站请求伪造（CSRF） | 利用用户已经通过认证的身份在未授权的网站上执行操作 |
| 文件包含漏洞 | 通过Web应用程序加载并执行恶意文件，如本地文件包含漏洞 |
| 拒绝服务攻击（DoS/DDoS） | 通过占用大量系统资源或发送大量请求来阻止合法用户访问服务 |
| 敏感信息泄露 | 通过未经授权的访问或不当配置来泄露用户或系统的敏感信息 |
| 点击劫持 | 将受害者的点击操作重定向到恶意网站上执行恶意操作 |


常见的Web攻击可以归纳为以下几个主要类别：

#### 跨站脚本攻击（XSS, Cross-Site Scripting）
    - 存储型XSS：攻击者将恶意脚本注入到Web应用程序的数据库中，当其他用户访问包含恶意脚本的数据时，脚本会在他们的浏览器中执行。
    - 反射型XSS：攻击者构造一个包含恶意脚本的URL，诱使受害者点击，当URL加载时，恶意脚本在受害者的浏览器中执行。
    - DOM-Based XSS：基于DOM环境的XSS，攻击发生时，恶意脚本由浏览器解析和执行，而不是服务器直接返回。



#### 跨站请求伪造（CSRF, Cross-Site Request Forgery）
    - 攻击者利用目标用户的已登录身份发起恶意请求，迫使用户在不知情的情况下执行非自愿的操作，如转账、更改密码等。



#### SQL注入（SQLi, SQL Injection）
    - 攻击者通过在Web表单字段或URL参数中输入精心构造的SQL命令，以期让Web应用程序误将攻击者提供的SQL代码当作有效查询执行，从而窃取、修改或删除数据库中的敏感信息。



#### 点击劫持（Clickjacking）
    - 攻击者创建一个透明或不可见的iframe，将目标网站的内容嵌入其中，欺骗用户点击实际位于下层页面的按钮或链接，以实现攻击者的目的。



#### URL跳转漏洞攻击
    - 也称为开放重定向，攻击者利用Web应用未能正确验证重定向地址的功能，将用户重定向至恶意网站。



#### 拒绝服务攻击（DoS/DDoS, Denial of Service/Distributed Denial of Service）
    - 攻击者通过大量的无效请求或流量耗尽服务器资源，使其无法正常响应合法用户的请求。



#### 会话劫持（Session Hijacking）
    - 攻击者通过各种方法获取用户的会话ID，从而假冒合法用户进行操作。



#### 文件包含（File Inclusion）
    - 攻击者通过注入恶意文件路径，使服务器错误地包含并执行外部文件，导致敏感信息泄露或远程代码执行。



#### HTTP头部注入（Header Injection）
    - 攻击者通过在HTTP请求头中插入恶意内容，影响服务器响应或进行其他形式的攻击。



#### 目录遍历（Traversal Attacks）
    - 攻击者尝试访问服务器上的非公开目录和文件。
