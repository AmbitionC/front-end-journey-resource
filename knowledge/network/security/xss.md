**XSS（Cross-Site Scripting，跨站脚本攻击）**是一种常见的网络安全漏洞，攻击者通过在网页中注入恶意脚本，使之在用户浏览器中执行，从而盗取用户数据、操控用户账户或执行恶意操作。XSS攻击主要分为三类：

#### 反射型XSS（Non-Persistent XSS）： 
    - 攻击方式：攻击者构造包含恶意脚本的URL，诱导受害者点击。服务器收到请求后，未对用户提交的数据进行充分过滤或编码，将恶意脚本原样回显到网页中，导致浏览器执行。
    - 防御措施： 
        * 对所有用户输入进行严格的输入验证和适当的编码，比如使用HTML实体编码或JavaScript转义。
        * URL重写或参数过滤，对危险字符进行转义或替换。
        * 不信任任何从客户端接收到的数据，即使是查询字符串参数也应该进行净化处理。



#### 存储型XSS（Persistent XSS）： 
    - 攻击方式：攻击者将恶意脚本注入到Web应用的数据库或其它持久化存储中，当其他用户访问含有恶意脚本的页面时，脚本被执行。
    - 防御措施： 
        * 同样的，对所有存储在服务器端的数据进行严格的安全编码。
        * 在显示存储内容到网页时，对数据进行输出编码，例如使用`textContent`而非`innerHTML`插入内容到DOM中。
        * 使用内容安全策略（Content Security Policy, CSP）来限制浏览器只执行可信来源的脚本。



#### DOM-Based XSS（基于DOM的XSS）： 
    - 攻击方式：恶意脚本并非由服务器直接输出，而是在客户端的DOM中被动态生成时产生，可能是由于客户端JavaScript处理用户输入不当导致。
    - 防御措施： 
        * 对所有的动态生成DOM内容进行严格的输入验证和输出编码。
        * 使用安全的DOM操作API，避免直接拼接字符串生成HTML内容。
        * 强化客户端的输入验证，尤其是在使用AJAX进行动态内容更新时。



综合防护手段还包括：

+ 使用HTTP-only属性保护Cookie，防止通过JavaScript读取和修改Cookie。
+ 教育用户警惕点击可疑链接，不轻易复制粘贴未经核实的URL到浏览器地址栏。
+ 对于复杂的应用场景，可以使用Web安全框架和库，它们通常内置了预防XSS攻击的机制。
