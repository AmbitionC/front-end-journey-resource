| 步骤 | 描述 |
| :---: | --- |
| 解析URL | 浏览器解析输入的URL，包括协议、域名等 |
| DNS解析 | 获取域名对应的IP地址 |
| 建立TCP连接 | 建立与目标服务器的TCP连接 |
| 发起HTTP请求 | 向服务器发送HTTP请求 |
| 服务器处理请求 | 服务器接收到请求并处理 |
| 返回HTTP响应 | 服务器向浏览器发送HTTP响应 |
| 浏览器渲染页面 | 浏览器接收响应数据并渲染页面 |
| 连接关闭 | 页面加载完成后，关闭与服务器的TCP连接 |


当用户在浏览器地址栏输入URL并按下回车键后，浏览器会执行一系列复杂的过程来显示网页内容。以下是浏览器处理URL并最终呈现页面的详细步骤：



####  URL解析: 
    - 浏览器首先检查输入的字符串是否符合URL的标准格式，包括协议（如`http://`或`https://`）、主机名（如`www.example.com`）和路径（如 `/path/to/page.html`）等组成部分。
    - 如果没有明确指定协议，浏览器通常会尝试添加 `http://` 并尝试访问，但如果站点只支持HTTPS，浏览器会重定向至HTTPS版本。



####  协议处理: 
    - 根据URL中的协议部分决定如何处理请求，如果是HTTPS协议，则会进行加密传输。



####  缓存检查: 
    - 浏览器会检查本地缓存，看看是否已经存储了该网址的最新版本内容。如果有且内容未过期，浏览器可以直接从缓存中加载页面而无需发送网络请求。



####  DNS解析: 
    - 如果需要向服务器请求新的数据，浏览器会首先查找本地DNS缓存（包括浏览器缓存、操作系统缓存、路由器缓存等）来获取与URL主机名关联的IP地址。
    - 如果缓存中没有记录或记录已过期，浏览器会向DNS服务器发送查询请求来解析域名。



####  TCP连接: 
    - 在获得域名对应的IP地址后，浏览器通过TCP/IP协议栈与服务器建立TCP连接。对于HTTP而言，默认端口是80（HTTP）或443（HTTPS）。
    - 这一步包括著名的“三次握手”过程来初始化连接。



####  SSL/TLS握手（仅HTTPS）: 
    - 如果是HTTPS连接，还会进行SSL/TLS握手以建立加密的连接，包括交换证书、验证服务器身份、协商加密算法和生成会话密钥等步骤。



####  发送HTTP请求: 
    - 连接建立完成后，浏览器构造并发送HTTP请求报文到服务器。请求报文包含了请求方法（GET、POST等）、URL、HTTP头信息（如Cookie、User-Agent等）以及可能的请求体数据。



####  服务器处理请求: 
    - 服务器接收请求，解析请求报文，执行必要的操作（如读取数据库、执行脚本等），并准备响应内容。



####  服务器发送HTTP响应: 
    - 服务器将HTTP响应报文发回给浏览器，响应报文包含状态码、响应头信息以及响应主体内容（如HTML、CSS、JavaScript等）。



####  浏览器解析响应: 
    - 浏览器接收到响应后，根据响应的内容类型开始解析响应数据，如渲染HTML文档，下载相关的CSS、JavaScript文件和图片等资源。



####  页面渲染: 
    - 浏览器根据HTML构建DOM树，根据CSS构建CSSOM树，两者合并形成渲染树，并根据渲染树计算布局（Layout），最后进行绘制（Painting），从而将网页内容呈现在屏幕上。



#### 执行JavaScript: 
    - 如果页面包含JavaScript，浏览器将在适当的时候执行这些脚本，它们可能会改变DOM结构、执行异步请求或触发其他动态行为。



#### 后续交互与事件处理: 
    - 页面加载完毕后，浏览器继续监听用户与页面的交互，执行相应的JavaScript代码来处理各种事件。
