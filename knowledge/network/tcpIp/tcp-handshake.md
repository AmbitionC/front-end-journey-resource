| 步骤 | 客户端行为 | 服务器行为 |
| --- | --- | --- |
| 第一步 | 客户端向服务器发送SYN包，建立连接请求 | 监听到SYN包，回复客户端一个SYN和ACK的包 |
| 第二步 | 客户端接收到服务器的SYN和ACK包，发送一个ACK包 | 接收到客户端的ACK包，连接建立成功 |
| 第三步 | - | 完成连接建立 |


TCP（Transmission Control Protocol）三次握手过程是用来建立一个可靠连接的关键步骤，确保客户端和服务端之间的数据传输通道是可靠和双向的。以下是详细的三次握手过程：



#### 第一次握手： 
    - 客户端（Client）主动发起连接请求，向服务器（Server）发送一个`SYN`（同步序列编号 Synchronize Sequence Numbers）报文段，其中包含一个随机产生的初始化序列号 `ISN_Client`（记为 X）。

```plain
Client -> Server: SYN = 1, SEQ = X
```

    - 此时，客户端进入`SYN_SENT`状态，等待服务器的确认。



#### 第二次握手： 
    - 服务器接收到客户端的`SYN`报文后，确认客户端的请求是有效的，同时也想与客户端建立连接，于是服务器会回应一个`SYN`和`ACK`（Acknowledgement，确认）标志位都置为1的报文段。
    - 服务器同样生成一个随机的初始化序列号 `ISN_Server`（记为 Y），并将确认号 `ACK_Num` 设置为客户端的序列号加1（即 X + 1）。

```plain
Server -> Client: SYN = 1, ACK = 1, SEQ = Y, ACK_NUM = X + 1
```

    - 服务器发送完此报文段后，进入`SYN_RECEIVED`状态。



#### 第三次握手： 
    - 客户端收到服务器的`SYN+ACK`报文后，验证服务器的确认号是否正确（即`ACK_NUM`等于先前发送的序列号加1），并且确认标志位`ACK`是否为1。
    - 如果确认信息无误，客户端再次向服务器发送一个确认报文，其确认号设置为服务器的序列号加1（即 Y + 1），并且自己的序列号递增（记为 X'）。

```plain
Client -> Server: ACK = 1, SEQ = X', ACK_NUM = Y + 1
```

    - 当服务器接收到这个确认报文后，确认客户端已经收到它的确认信息，且客户端也准备好进行数据传输，此时双方都进入`ESTABLISHED`状态，至此，TCP连接建立完成，可以开始互相发送数据。

整个三次握手过程确保了双方都知道彼此的初始序列号，同时也确认了双方都有发送和接收数据的能力。通过这样的机制，TCP能够在不可靠的网络环境下建立一个可靠的连接。
