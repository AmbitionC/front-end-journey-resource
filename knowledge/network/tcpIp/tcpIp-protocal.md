| 层级 | 功能 | 协议 |
| :---: | --- | --- |
| 应用层 | 提供用户接口和服务，与用户直接交互 | HTTP、FTP、SMTP、DNS等 |
| 传输层 | 提供端到端的数据传输服务，确保可靠性和顺序性 | TCP、UDP |
| 网络层 | 负责数据在网络中的传输和路由 | IP、ICMP、ARP等 |
| 数据链路层 | 将数据包封装成帧，在相邻节点之间传输 | Ethernet、PPP、ATM等 |
| 物理层 | 确定传输介质上的数据的电气特性 | RJ45、光纤等 |




TCP/IP协议栈是一个四层模型，它定义了电子设备如何连入因特网，以及数据在网络中如何传输的标准。这个模型基于OSI七层模型简化而来，主要包括以下四个层次：

#### 应用层 (Application Layer) 
    - 这是最顶层，直接与应用程序交互，为应用程序提供服务。
    - 常见协议包括HTTP、HTTPS、FTP、SMTP（电子邮件）、DNS（域名系统）、SSH（安全外壳）等。



#### 传输层 (Transport Layer) 
    - 主要负责端到端的数据传输，确保数据的可靠传输或者无连接的数据报传输。
    - 两个重要的传输协议是： 
        * TCP（Transmission Control Protocol，传输控制协议）：提供面向连接的服务，保证数据的有序、可靠传输。
        * UDP（User Datagram Protocol，用户数据报协议）：提供无连接的服务，不保证数据的顺序和可靠性，但具有较低的开销和较高的实时性。



#### 网络层 (Network Layer) 
    - 负责主机之间的通信，实现数据包从源到目的地的路由选择。
    - 最核心的协议是IP（Internet Protocol，互联网协议），负责为数据包提供逻辑地址（即IP地址）并决定最佳路径传输数据。



#### 链路层 (Link Layer) 或 数据链路层 
    - 有时也称为网络接口层，在不同网络之间提供节点间的数据传输。
    - 包括各种子层和协议，例如以太网协议（Ethernet）、Wi-Fi协议（802.11）和PPP（Point-to-Point Protocol）等，它们负责物理寻址、错误检测和纠正，以及数据帧的封装与解封装。



每一层都为下一层提供服务，并使用下一层提供的服务，这样构建起来的分层结构使得复杂的网络通信问题得以分解解决，并且能够灵活适应不同的网络技术和硬件变化。
