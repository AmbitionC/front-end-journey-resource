以下是TCP报文格式的概述：

| 字段 | 长度（字节） | 描述 |
| :---: | :---: | --- |
| 源端口 | 2 | 发送端口号 |
| 目标端口 | 2 | 接收端口号 |
| 序列号 | 4 | 数据的序列号，用于排序和重组数据包 |
| 确认号 | 4 | 下一个期望接收的数据包的序列号 |
| 数据偏移 | 4 | 表示TCP头部长度的字段，通常为4个32位字（即20字节） |
| 控制位 | 2 | 用于控制TCP连接和数据传输的标志位 |
| 窗口大小 | 2 | 接收窗口的大小，用于流量控制 |
| 校验和 | 2 | 用于校验TCP头部和数据的完整性 |
| 紧急指针 | 2 | 用于指示紧急数据的位置 |
| 选项 | 可变 | 可选字段，用于传递一些特定的参数和选项 |
| 数据 | 可变 | 实际的传输数据，长度取决于MTU和网络条件 |
