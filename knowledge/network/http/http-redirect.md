**永久性重定向（301 Redirect）与暂时性重定向（302 Redirect）的区别**如下：

| 特性 | 永久性重定向 (301) | 暂时性重定向 (302) |
| :---: | --- | --- |
| HTTP状态码 | 301 Moved Permanently | 302 Found |
| 定义 | 资源已永久移动到新URL | 资源暂时在新URL下可用 |
| SEO权重转移 | 是，权重完全从旧URL传递到新URL | 否，或者不确定，搜索引擎可能不转移权重 |
| 应用场景 | 网站改版、域名更换、路径重构 | 页面维护、临时跳转、未确定的将来还会更改回原URL的情况 |
| 缓存策略 | 浏览器和其他客户端可能缓存结果 | 理论上不被缓存，但实际中某些客户端可能仍会缓存 |
| 长期效果 | 对SEO友好，有利于保持或提高排名 | 不利于SEO，可能导致搜索引擎索引混乱 |


#### HTTP状态码定义：
+  **301 Moved Permanently（永久移动）**：当服务器返回301状态码时，意味着资源已被永久性地移动到了新的URL，任何未来对该资源的引用都应该使用新的URL。搜索引擎会把此信号视为旧URL已经失效，并将旧URL的权重转移到新URL上。 
+  **302 Found（临时移动）**：302状态码表示资源目前在不同的URL下临时可用，但未来的请求仍然可能被指向原来的URL。搜索引擎通常不会将此状态码视为永久迁移，并且不会立即把旧URL的权重传递给新URL。 



#### 应用场景及影响：
+  **301永久重定向**： 
    - 适用于网站改版、域名更换、网页路径重构等情况，有助于保持SEO排名和流量不流失。
    - 浏览器和搜索引擎都会记住这个重定向，并在之后的请求中自动使用新的URL。
+  **302暂时重定向**： 
    - 主要用于临时性的场景，如页面正在维护、临时跳转至登录页面等。
    - 搜索引擎一般不会将302重定向视作永久变化，因此不会立刻将链接流行度和排名从原URL转移到新URL。
    - 用户每次访问时，除非服务器明确指示，否则浏览器不会默认使用之前重定向的新URL。



#### 缓存策略：
+ **301重定向**：浏览器和其他客户端可能会缓存永久重定向的结果。
+ **302重定向**：理论上讲，临时重定向不应被客户端缓存，但在实际应用中，某些老旧或非标准的客户端可能也会缓存302响应。



#### SEO考量：
+ **301重定向**：对于SEO友好，因为它允许搜索引擎将旧页面的所有“链接 juice”（即链接权重）传递到新页面，从而保留或提升搜索排名。
+ **302重定向**：不适合长期使用或在需要传递权重的情况下使用，因为不保证搜索引擎会将权重传递到目标URL。
