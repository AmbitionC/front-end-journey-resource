基于HTTP的功能追加协议是指那些在HTTP协议的基础上，为了增强功能或改进性能而设计的一系列扩展协议。以下是一些常见的基于HTTP的功能追加协议：

#### Ajax (Asynchronous JavaScript and XML) 
    - Ajax并非一个新的网络协议，而是一种使用现有HTTP协议实现网页异步更新的技术手段。通过JavaScript和XMLHttpRequest API，可以在后台与服务器交换数据并局部刷新网页内容，极大地改善了用户体验。



#### Comet 
    - Comet是一种让服务器能够主动向客户端推送数据的技术集合，突破了HTTP传统的请求-响应模型。通过长时间保持HTTP连接打开或者周期性轮询的方式，实现了服务器到客户端的实时数据推送。



#### SPDY（现已废弃，功能合并至HTTP/2） 
    - SPDY旨在减少网页加载延迟，提高网络应用性能。它引入了多路复用、优先级控制、压缩HTTP头部以及支持TLS加密等特性，后来成为了HTTP/2协议的重要基础。

#### HTTP/2 
    - HTTP/2是HTTP协议的一个重要升级版本，它吸取了SPDY的经验，实现了多路复用、头部压缩、服务器推送等功能，大幅提升了HTTP的性能和效率。

#### WebSocket 
    - WebSocket协议是一种全双工通信协议，它在HTTP握手之后建立了一个持续的双向通信通道，使得服务器与客户端之间可以实时、双向地交换数据，常用于实时聊天、游戏、股票报价等场景。

#### Server-Sent Events (SSE) 
    - SSE是一种允许服务器向浏览器发送实时更新的标准机制，它通过HTTP连接单向推送事件，简化了服务器向客户端发送更新的过程。

#### QUIC（Quick UDP Internet Connections） 
    - QUIC是在UDP之上构建的多路复用传输层协议，旨在降低延迟并提高网络应用性能。虽然不是直接基于HTTP，但它支持HTTP/3，后者是HTTP协议在QUIC上的实现版本。
