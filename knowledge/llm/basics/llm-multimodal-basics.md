多模态模型不是简单地把图片、声音和文字“塞进同一个输入框”。不同模态先由适合自己的编码器转成数值表示，再通过对齐或融合机制进入共享模型；输出端也可能需要专门的解码器。理解这条数据路径，才能判断一个模型究竟“看见”或“听见”了什么。

## 什么是模态

模态是信息的表现与感知形式，例如文本 token、图像像素、音频波形和视频帧。它们的结构不同：文本是离散序列，图像具有二维空间关系，音频强调连续时间变化，视频同时包含空间与时间。

因此，多模态系统通常要解决三个问题：

1. **编码**：怎样从原始信号提取可学习的表示；
2. **对齐与融合**：怎样建立“这段文字对应这块图像/这段声音”的关系；
3. **生成或决策**：怎样把融合后的表示变成文本、图像、音频或动作。

## 一条典型的多模态流水线

![文本图像音频经过专用编码器和跨模态融合进入生成模型](https://font-end-journey-resources.oss-cn-hangzhou.aliyuncs.com/images/llm-multimodal-basics-fusion-pipeline-v1.webp)
*图：原始模态先经过 tokenizer 或专用编码器，再以 embedding/token 形式融合；具体系统支持哪些输出，取决于是否配置了相应输出头或解码器。*

### 1. 模态编码

- 文本经过 tokenizer 变成 token ID，再映射为 embedding。
- 图像常被切成 patch，由视觉编码器提取空间特征。
- 音频可先转成频谱等时频表示，再由音频编码器处理。

[CLIP](https://arxiv.org/abs/2103.00020) 使用独立的图像编码器和文本编码器，通过对比学习让匹配的图文表示靠近、不匹配的表示远离。这证明了自然语言可以作为大规模视觉监督，但 CLIP 本身主要学习可比较的表示，并不是一个开放式文本生成器。

### 2. 对齐与融合

常见方案包括：

- **共享表示空间**：把不同模态投影到可比较的向量空间，适合检索和分类；
- **跨注意力**：文本查询选择视觉或音频特征，适合问答和条件生成；
- **统一 token 序列**：把模态特征离散化或投影成模型可处理的 token；
- **晚期融合**：不同模型先独立判断，再在分数或决策层组合。

[Flamingo](https://arxiv.org/abs/2204.14198) 展示了一种连接预训练视觉编码器与语言模型的方式：视觉特征经重采样后，通过门控 cross-attention 注入语言模型，从而处理交错的图像和文本。这是一种架构实例，不是所有多模态模型的唯一做法。

### 3. 输出解码

语言模型天然输出文本 token。若系统要生成图像或语音，还需要图像扩散/自回归解码器、声码器或统一的多模态 token 解码机制。界面上能上传图片，不代表模型必然能生成图片；能转录音频，也不代表它理解所有非语音声学事件。

## 三类核心任务

### 理解与检索

图文检索、图片分类、音频事件识别都可以归结为“把输入映射到表示并比较”。这类任务输出空间明确，容易建立离线指标。

### 条件生成

视觉问答、图片描述、语音转写需要让一个模态约束另一个模态的生成。[Whisper](https://arxiv.org/abs/2212.04356) 以大规模弱监督音频—文本数据训练 encoder-decoder，可执行多语言转写与翻译；它说明音频模型也能形成通用基础能力，但目标仍主要是语音到文本。

### 跨模态行动

Agent 可以读取截图、听取语音、结合文本指令后调用工具。此时真正的系统能力等于“感知模型 + 推理模型 + 工具权限 + 状态管理”，不能只用一个多模态基准分数概括。

## 一个文档助手示例

用户上传一张带图表的 PDF 并提问时，可靠流程可以是：

1. 文档解析器提取文本块、页面坐标和图片；
2. OCR 处理图片中的文字，视觉编码器提取图表结构；
3. 检索器按问题召回相关页面；
4. 融合模块把文字证据与视觉区域关联；
5. 模型生成带页码和区域引用的答案；
6. 对关键数字再用表格解析或计算工具核对。

如果直接把整份 PDF 截成图片输入，细小文字可能因缩放丢失，页间关系也会变弱。多模态输入并不免除解析和检索工程。

## 主要限制

- **空间与计数**：模型可能识别对象，却弄错左右、遮挡、数量或细粒度位置。
- **分辨率与采样**：图像缩放、视频抽帧、音频窗口都会丢信息。
- **模态冲突**：字幕、画面和声音矛盾时，模型可能偏向训练中更强的模态。
- **时间同步**：音画对齐、长视频事件顺序比单帧问答更难。
- **数据与安全**：图片可能包含个人信息，音频可能涉及生物特征，外部文档还可能携带间接提示注入。
- **输出可验证性**：自然语言解释仍可能虚构视觉细节，需要区域引用或专用检测器验证。

## 选型检查表

不要只问“是否多模态”，还应确认：支持哪些输入和输出、单次可处理的分辨率/时长、是否保留坐标和时间戳、是否支持多图/视频、能否返回引用、数据是否会被存储，以及关键任务用什么真实样本评估。

## 小结

多模态模型的核心是编码、对齐/融合与解码。不同模态不会自动共享语义，系统也不会因为能接收媒体就获得可靠感知。把各阶段拆开评估，才能知道错误来自输入解析、表示对齐、语言生成还是外部工具。
