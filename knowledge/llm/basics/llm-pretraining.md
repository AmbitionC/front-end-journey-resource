预训练的核心任务看似简单：根据前文预测下一个 token。真正困难的是在有限计算预算内，同时分配模型参数、训练 token 和数据质量，让损失下降能转化为可迁移能力，而不是记住重复内容或评测答案。

## 预训练到底在优化什么

对 token 序列 $x_1,\ldots,x_T$，自回归语言模型最小化交叉熵：

$$
\mathcal{L}=-\sum_{t=1}^{T}\log P_\theta(x_t\mid x_{<t})
$$

训练时使用 teacher forcing：预测第 $t$ 个 token 时能看到真实前缀。推理时模型看到的是自己先前生成的 token，因此早期错误可能继续影响后续。

这个目标不直接标注“事实”“推理步骤”或“安全回答”，但在足够多样的数据和容量下，模型会为了更好预测而学习语法、世界关联、代码结构和上下文模式。学到什么取决于数据分布，不等于建立完整、可更新的世界模型。

## 参数、数据与计算的三角约束

![预训练中模型参数训练token和计算预算的平衡关系](https://font-end-journey-resources.oss-cn-hangzhou.aliyuncs.com/images/llm-pretraining-scaling-triangle-v1.webp)
*图：固定计算预算下，扩大参数会减少可训练 token，增加 token 又要求足够模型容量；数据质量贯穿整个分配问题。*

[Scaling Laws for Neural Language Models](https://arxiv.org/abs/2001.08361) 观察到语言模型验证损失会随参数、数据量和训练计算呈经验幂律变化。这类规律可用于小规模试验外推，但它描述特定架构与数据条件下的趋势，不是保证能力按固定比例增长的自然定律。

[Chinchilla 研究](https://arxiv.org/abs/2203.15556) 指出，当时许多大模型在给定计算预算下参数过多、训练 token 不足；其试验支持同时扩大模型与数据。工程上的结论是“联合规划参数与 token”，而不是机械套用一个永远正确的 tokens-per-parameter 数字。数据质量、重复率、上下文长度、架构和推理成本都会改变最优点。

## 数据流水线决定模型看到的世界

### 收集与许可

来源可能包括网页、书籍、代码、论文、对话和授权数据。团队需要记录许可、隐私、地域与使用限制；“公开可访问”不自动等于“可任意训练和再分发”。

### 质量过滤

过滤器会去除模板垃圾、乱码、恶意页面和低信息内容，并提高目标语言、代码或专业资料的采样权重。过滤器本身也有偏差：过严可能删除方言与少数群体内容，过松则放大垃圾和攻击语料。

### 去重

精确去重移除完全相同文档，近似去重识别轻微改写或模板重复。去重能减少记忆、避免高频副本支配梯度，也有助于降低训练集与评测集污染。

### 数据混合

不同来源不是简单按原始体量拼接。代码、数学、多语言和高质量长文通常需要显式采样权重。混合比例改变模型能力，也可能造成训练后期遗忘或风格偏移。

## 一次训练如何运行

1. 文档经 tokenizer 转成 token 并构造成训练序列；
2. 模型前向计算每个位置的 logits 与 loss；
3. 反向传播得到梯度；
4. 优化器更新参数，学习率按 warmup、稳定期和衰减计划变化；
5. 周期性保存 checkpoint，并在保留集上评估；
6. 监控 loss spike、梯度范数、吞吐、硬件错误和数据异常。

大训练会组合数据、张量、流水线和上下文并行。有效 token 吞吐不仅受 GPU 算力影响，还受网络通信、序列长度分布、checkpoint 和故障恢复影响。

## 继续预训练与长上下文扩展

基础预训练后，可在目标领域或更长序列上继续训练。领域继续预训练能加强术语和文体，但比例过高可能损伤通用能力。长上下文扩展也不只是修改一个最大长度参数：位置表示、长序列数据、训练稳定性和“能放进去但能否利用”的评估都要同步改变。

[Llama 3 技术报告](https://arxiv.org/abs/2407.21783) 展示了数据质量模型、退火数据和长上下文继续预训练等组合策略，说明现代预训练通常由多个受控阶段构成。

## 怎样判断预训练是否成功

至少同时观察：

- 训练与验证 loss 是否稳定下降，二者差距是否异常；
- 不同数据域和语言的分桶 loss；
- 保留能力集、代码/数学和长上下文测试；
- 记忆复现、隐私、毒性与污染检查；
- 相同计算或推理预算下与基线的比较。

基准提升不应只报告最优一次运行。提示格式、few-shot 示例、解码方式和数据污染都可能改变结果。

## 常见误区

- **更多 token 一定更好**：重复或低质 token 可能浪费计算并强化错误模式。
- **更大参数一定更划算**：部署成本、训练 token 和计算预算共同决定可行规模。
- **训练数据越新越好**：新鲜度重要，但来源可靠性、许可和时间一致性同样重要。
- **预训练完成后模型就会听指令**：next-token prediction 与助手行为不是同一目标，仍需后训练。
- **Scaling Law 能预测所有下游能力**：loss 趋势比具体涌现能力更稳定，外推需要保留不确定性。

## 小结

预训练是一个受计算预算约束的数据—模型联合优化问题。高质量、去重且比例合理的数据，合适的参数/token 分配，稳定的分布式训练和独立评估，缺一不可。Scaling Law 是规划工具，不是跳过实验和风险判断的公式。
