大模型量化用较少比特表示权重或运行时张量，以降低显存占用、存储和内存带宽压力。它不是简单把文件“压缩一下”：数值表示、校准数据、硬件内核和任务敏感度共同决定速度与质量。

## 从浮点数到低比特整数

最基本的均匀量化用 scale $s$ 和 zero point $z$ 把浮点数 $x$ 映射到有限整数范围：

$$
q=\operatorname{clip}(\operatorname{round}(x/s)+z, q_{min}, q_{max})
$$

计算时再用 $\hat{x}=s(q-z)$ 近似原值。量化误差是 $x-\hat{x}$。位宽越低，可表示的级别越少；异常大值还会拉宽范围，让大多数普通值共享更粗的刻度。

工程实现常把张量分组，每组独立保存 scale，有时使用对称量化（$z=0$）。较小 group size 通常能更贴合局部分布，但会增加 scale 元数据与内核复杂度。

## 量化对象不只有权重

![权重、激活与 KV Cache 量化在显存、速度和精度之间的权衡](https://font-end-journey-resources.oss-cn-hangzhou.aliyuncs.com/images/llm-quantization-memory-accuracy-v1.webp)
*图：降低位宽通常减少内存和带宽需求，但收益依赖硬件内核，误差也会随量化对象与任务而变化。*

- **权重量化（W8/W4）**：主要减少模型权重体积，最常用于单卡或边缘部署。
- **激活量化（A8/A4）**：还能降低计算与中间张量带宽，但激活随输入变化，校准更难。
- **KV Cache 量化**：长上下文和大批量推理时很重要；它不改变权重大小，却能降低随序列长度增长的缓存成本。

“W4A16”通常表示 4-bit 权重、16-bit 激活；具体格式还可能包含 per-channel/per-group、对称性、累加精度和是否保留部分层为高精度，不能只看一个位数。

## PTQ 与 QAT

**训练后量化（PTQ）**在模型训练完成后量化，通常只需少量校准样本甚至无需训练，成本低、适合快速部署。**量化感知训练（QAT）**在训练中模拟量化误差，让参数适应低精度，潜在质量更好，但需要训练数据和更多计算。

[GPTQ](https://arxiv.org/abs/2210.17323) 是权重 PTQ 方法，逐层量化时利用近似二阶信息补偿误差；论文展示了在较低权重位宽下对大型生成模型进行一次性量化。[AWQ](https://arxiv.org/abs/2306.00978) 观察激活来识别少量重要权重，并通过逐通道缩放保护它们，重点在部署友好的 4-bit 权重量化。二者名称不代表统一文件格式，实际兼容性取决于实现和推理引擎。

## 显存账怎样算

只算权重时，$N$ 个参数、每参数 $b$ bit 的理想下界约为：

$$
\text{weight bytes}\approx N\times b/8
$$

因此 70 亿参数的 FP16 权重约 14 GB，INT4 理想值约 3.5 GB。这只是估算：真实占用还包括 scale、zero point、未量化层、内存对齐、运行时 workspace、激活和 KV Cache。训练还需梯度与优化器状态，不能沿用纯推理权重账单。

## 量化一定更快吗

不一定。量化最稳定的收益通常是减少内存容量与带宽；速度还取决于：

- GPU、CPU 或 NPU 是否原生支持该数据类型；
- 推理框架是否有匹配的 fused kernel；
- 解包、反量化开销是否被矩阵计算抵消；
- batch、序列长度和模型是否处于带宽瓶颈；
- 张量并行通信能否利用更小表示。

某些消费级设备上的 4-bit 模型可以更快，另一些硬件可能因转换开销不如优化良好的 FP16/BF16。必须在目标设备和真实请求分布上压测。

## 怎样做质量验收

量化误差对任务并不均匀。困惑度变化小，不保证数学、代码、长上下文或工具参数同样稳定。建议：

1. 固定原模型、提示、解码参数和评测数据；
2. 比较任务成功率、生成质量、延迟、吞吐和峰值显存；
3. 单独检查长上下文、少数语言、数字与结构化输出；
4. 对敏感层、输出头或 embedding 尝试保留更高精度；
5. 记录量化算法、位宽、group size、校准集和运行时版本。

[Llama 3 技术报告](https://arxiv.org/abs/2407.21783) 也强调部署精度与训练/推理栈需要联合验证；模型规模相同并不意味着不同量化产物可互换。

## 常见误区

- **INT4 就一定省 75% 全部显存**：它只给出权重的理论比例，运行时还有其他占用。
- **文件能加载就代表质量合格**：格式兼容不等于任务指标无回归。
- **校准数据随便取**：激活分布与目标业务不匹配会放大量化误差。
- **低 perplexity 等于没有损失**：特定能力可能先退化。
- **同一“4-bit”都一样**：算法、分组、量化对象和内核都可能不同。

## 小结

量化是在容量、带宽、速度与精度之间重新分配预算。先明确量化权重、激活还是 KV Cache，再选择 PTQ/QAT 与格式，最后在目标硬件上同时测质量和性能。位宽是起点，不是结论。
