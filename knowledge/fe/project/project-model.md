以下是几种基础的设计模式：

| 模式 | 描述 |
| :---: | --- |
| 工厂模式 | 用于创建对象的模式，通过一个工厂方法来创建对象，隐藏了对象的具体实现细节，提供了统一的接口用于对象的创建。 |
| 单例模式 | 保证一个类只有一个实例，并提供全局访问点，常用于管理共享资源、配置信息等场景，如全局缓存、日志记录等。 |
| 观察者模式 | 也称为发布-订阅模式，定义了一种一对多的依赖关系，当对象的状态发生变化时，所有依赖它的对象都会收到通知并自动更新。 |
| 装饰者模式 | 允许向一个对象动态地添加新功能，通过将对象包装在一个装饰器中，并在运行时添加额外的行为，而不是在编译时修改代码。 |
| 策略模式 | 定义一系列算法，并将每个算法封装到一个类中，使得算法的变化可以独立于使用它的客户端，使得算法可以互换和替换。 |
| 原型模式 | 用于创建对象的模式，通过复制现有对象来创建新对象，提供了一种创建对象的方法，避免了使用类和构造函数。 |
