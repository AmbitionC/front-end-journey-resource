下表是Less、Sass（SCSS）和CSS的差异：

| 特性/方面 | CSS | Less | Sass (SCSS) |
| :---: | --- | --- | --- |
| **基础定义** | 用于描述HTML文档的样式和布局的语言 | 动态样式语言，扩展了CSS的功能 | 动态样式语言，扩展了CSS的功能，与Less类似，但有更强大的社区支持 |
| **语法** | 固定语法，使用花括号、分号和逗号 | 类似于CSS的语法，使用`@`<br/>符号定义变量和混合等 | 使用花括号、分号和逗号，与CSS相似，但缩进表示层级关系（旧版Sass）或使用与Less类似的语法（SCSS） |
| **变量** | 不支持变量，所有值必须直接定义 | 支持变量，使用`@varname: value;` | 支持变量，使用`$varname: value;` |
| **混合（Mixins）** | 不支持混合，无法创建可重用的样式块 | 支持混合，使用`.mixin()` | 支持混合，使用`@mixin` |
| **函数** | 内置有限的CSS函数 | 支持自定义函数，使用`@function` | 支持自定义函数，使用`@function` |
| **操作** | 不支持颜色和数值的运算 | 支持颜色和数值的运算 | 支持颜色和数值的运算 |
| **导入** | 不支持导入其他样式表 | 支持导入Less文件，使用`@import` | 支持导入Sass/CSS文件，使用`@import` |
| **模块化** | 不支持模块化 | 支持模块化，有助于代码组织 | 支持模块化，有助于代码组织 |
| **编译** | 不需要编译，直接被浏览器解析 | 需要编译成CSS | 需要编译成CSS |
| **浏览器支持** | 所有浏览器原生支持 | 需要编译后的CSS才能被浏览器支持 | 需要编译后的CSS才能被浏览器支持 |
| **预处理器特性** | 无 | 支持，如Less.js可以在客户端编译 | 支持，通常在服务器端或构建过程中编译 |
| **社区和工具** | 非常成熟，几乎所有网页设计都使用 | 有一定的社区支持，工具链较为完善 | 社区庞大，工具链非常强大，如Koala、Compass等 |
| **学习曲线** | 非常平缓，易于上手 | 相对平缓，适合快速上手 | 稍微陡峭，但提供了丰富的学习资源 |
