| 标签 | 描述 | 作用 |
| --- | --- | --- |
| <header> | 定义文档或节的页眉 | 标识页面顶部，通常包含导航、标题等内容 |
| <nav> | 定义导航链接的容器 | 标识页面导航，包含导航链接等内容 |
| <section> | 定义文档的某个区域 | 标识页面中的一个区域，通常包含一组相关内容 |
| <article> | 定义独立的文章或内容 | 标识页面中的一个独立内容块，如新闻、博客文章等 |
| <aside> | 定义侧边栏内容 | 标识页面侧边栏，通常包含与主内容相关的附加信息 |
| <footer> | 定义文档或节的页脚 | 标识页面底部，通常包含版权信息、联系信息等 |
| <main> | 定义文档的主要内容 | 标识页面的主要内容区域 |
| <figure> | 定义媒体内容的容器 | 标识图像、音频、视频等媒体内容及其标题或说明 |
| <figcaption> | 定义<figure>元素的标题或说明 | 用于标识图像、音频、视频等媒体内容的标题或说明 |




HTML5引入了一系列新的语义化标签，这些标签旨在更好地描述网页内容的结构，从而提高网页的可访问性和搜索引擎优化（SEO）。以下是一些常用的HTML5语义化标签及其作用：

+ `**<header>**`：定义文档或部分的页眉，通常包含标题、导航条、搜索表单等。它有助于告诉浏览器和搜索引擎这是页面的头部区域。 
+ `**<nav>**`：定义导航链接的容器，通常用于页面的主导航菜单。使用`<nav>`标签可以帮助搜索引擎理解页面的导航结构。 
+ `**<section>**`：表示文档中的一个独立部分，比如章节、页眉、页脚或任何从文档中分离出来的部分。`<section>`应该包含一个标题（`<h1>`到`<h6>`）。 
+ `**<article>**`：定义独立的内容，比如博客文章、新闻故事或论坛帖子。`<article>`内容应该是自包含的，即使从页面中移除也能独立存在。 
+ `**<aside>**`：定义与页面内容相关但可以独立于内容存在的部分，通常用作侧边栏，包含与主内容相关但不是直接部分的内容，如相关链接、广告等。 
+ `**<footer>**`：定义文档或部分的页脚，通常包含版权信息、相关链接、作者等。它有助于告诉浏览器和搜索引擎这是页面的底部区域。 
+ `**<figure>**`：用于包含媒介内容（如图像、图表、照片、代码等），并通过`<figcaption>`标签提供标题或说明。 
+ `**<figcaption>**`：定义`<figure>`元素的标题或说明，通常用于图注。 
+ `**<main>**`：表示文档的主体内容，一个页面上只能有一个`<main>`元素，它应该包含页面的核心内容。 
+ `**<address>**`：定义联系信息，如作者或拥有者的联系信息，通常用于页脚。 
+ `**<time>**`：表示特定的日期或时间。它可以通过`datetime`属性提供机器可读的日期或时间。 
+ `**<mark>**`：定义突出显示的文本，通常用于高亮显示搜索结果中的关键词。 



使用这些语义化标签的好处包括：

+ **提高可访问性**：辅助技术（如屏幕阅读器）可以利用这些标签更好地理解页面内容，从而为视障用户提供更好的体验。
+ **改善SEO**：搜索引擎可以更准确地理解页面结构和内容，从而可能提高网站的搜索排名。
+ **代码维护**：语义化标签使得代码更加清晰和易于维护，有助于开发者快速理解页面结构。
