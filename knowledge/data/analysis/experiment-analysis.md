实验分析不是先把数据扔进显著性检验，而是在看结果前定义问题：假设是什么、随机化单位是谁、希望估计哪个效应、主指标和护栏是什么、何时停止。预先定义能防止结果出来后不断换口径，直到找到一个好看的 p 值。

## 先定义 estimand

“新功能是否有效”过于模糊。应写成：在符合条件的用户中，分配到新版本对 7 日付费率的 intention-to-treat 平均效应。这里包含总体、处理、结果、时间窗和效应口径。

随机化单位可能是用户、设备、门店或集群。若按用户随机却按事件当独立样本，重度用户会被重复计算并低估标准误。社交传播、共享库存等干扰还会破坏“一个人的处理不影响另一个人”的假设，此时要按群组随机或明确解释局限。

## 指标体系

只选一个或少数 primary metric，直接对应决策；secondary metric 用于解释机制；guardrail 监控延迟、投诉、取消等不可接受损害。每个指标固定分母、去重、窗口、异常值和缺失处理。

在实验前运行 A/A，检查分流、埋点、Sample Ratio Mismatch 和指标方差。SRM 往往意味着随机化、曝光或过滤有 bug，不能通过统计修正掩盖。

## 效应量与置信区间

报告绝对差、相对差和基线。例如转化率从 10.0% 到 10.3%，绝对提升 0.3 个百分点，相对提升 3%。业务成本通常与绝对量相关，只报相对提升容易夸大。

[NIST 置信区间说明](https://www.itl.nist.gov/div898/handbook/prc/section1/prc14.htm)强调，频率学置信区间描述重复抽样程序的覆盖率，不是“固定真实参数有 95% 概率落在本次区间”。区间宽度表达当前设计下的不确定性；跨过 0 表示证据不足以排除无效，不等于证明两个版本完全相同。

[NIST 假设检验说明](https://www.itl.nist.gov/div898/handbook/prc/section1/prc15.htm)展示检验与区间的关系。p 值是在零假设与模型成立时得到至少同样极端数据的概率，不是零假设为真的概率。决策还应比较区间与最小有意义效应，而非只看 0.05。

## 功效、样本量与停止规则

实验前用基线率、最小可检测效应、显著性水平、功效和随机化单位估算样本。运行时间还要覆盖业务周期。样本不足时“没有显著”可能只是区间太宽；样本过大则会把无业务价值的小差异判为显著。

持续每天查看并在好看时停止会增加假阳性。采用固定样本/时间，或预先选择能支持连续监控的 sequential 方法。数据质量和严重安全护栏可以随时触发暂停，但这与为了 primary metric 提前获胜不同。

## 多重比较与事后分析

同时测试许多指标、变体和子群，至少一个偶然显著的概率会上升。[NIST 多重比较资料](https://www.itl.nist.gov/div898/handbook/prc/section4/prc47.htm)说明需要控制整体错误。可使用 Bonferroni、Holm 或 FDR 等方法，具体取决于确认性还是探索性目标。

结果后发现的“某地区特别有效”应标记探索性，并在新实验复现；不能装成预先假设。保留完整分析清单，包括未显著的指标，避免选择性报告。

## 缺失、异常与因果边界

按 treatment assignment 分析 ITT，避免只保留实际使用功能的人造成自选择偏差。曝光日志缺失、机器人流量和极端值处理在看结果前固定。若必须剔除，报告各组剔除率和敏感性分析。

随机实验估计的是当前样本、实现和时期下的效应。推广到节假日、新国家或长期留存需要额外证据。统计方法不能修复错误埋点、交叉污染或业务规则改变。

## 结果报告模板

报告包含假设、注册时间、总体与随机化单位、样本量、实验周期、SRM、主指标效应与区间、护栏、多重校正、数据排除、已知限制和建议。将分析代码、数据版本和 experimentId 固化，保证复算一致。

最终决策可以是上线、不上线、继续收集或重新设计。一个诚实的“区间仍包含有价值提升和有害下降，需要更多数据”，比把不确定结果硬解释成胜利更有价值。
