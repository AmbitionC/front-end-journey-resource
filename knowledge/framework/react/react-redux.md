Redux是JavaScript应用程序中广泛使用的状态管理库，它提供了一个集中式存储来管理应用程序的状态，并以相应的纯函数来执行状态的修改，以确保没有任何不可预测的副作用。Redux的设计思路基于几个核心原则：

![](https://cdn.nlark.com/yuque/0/2024/png/577681/1711864708538-28d08389-d859-4274-9ad4-100c9f5155b8.png)

#### 1. 单一数据源
整个应用程序的状态被存储在一个单一的对象树中，称为store。这使得状态的管理和调试变得更加容易，因为所有的状态变更都可以在一个地方找到。



#### 2. 状态是只读的
唯一改变状态的方法是触发action，这是一个描述发生什么的普通对象。Action对象中包含了导致状态变化的信息。



#### 3. 使用纯函数执行修改
为了描述action如何改变状态树，你需要编写reducer。Reducer是纯函数，它接收先前的状态和一个action，然后返回新的状态。纯函数没有副作用，不依赖于外部环境，只依赖于输入参数。



#### Redux工作流程
以下是Redux库处理状态变更的基本流程：

1.  **Action**：当用户与界面交互时（例如点击按钮），应用程序会创建一个action对象，该对象描述了发生了什么事情。 
2.  **Dispatch**：然后将这个action发送给store（通过dispatch方法），store运行reducer函数来处理这个action。 
3.  **Reducer**：Reducer根据当前state和给定的action计算出新的state。 
4.  **Store**：Store保存了应用程序的状态树。任何时候，你都可以调用store.getState()来获取当前的状态副本。 
5.  **View**：应用程序的视图层订阅store，当store中的状态发生变化时，视图层会接收到通知，并从store中获取新的state，然后更新UI。 



#### Redux的核心概念
+ **Actions**：描述应用状态变化的对象，包含type和可能的payload。
+ **Reducers**：纯函数，根据action计算出新的状态。
+ **Store**：承载应用状态的容器，任何状态变化都通过store进行。
+ **Dispatching**：通过dispatch方法发送action到store。
+ **Subscribing**：视图层订阅store，监听状态变化。



#### Redux的优势
+ **预测性**：由于reducer是纯函数，状态的变化变得可预测。
+ **可调试性**：所有的状态变化都是可追踪的，因为你可以查看action和reducer。
+ **模块化**：Action和reducer可以轻松地重用和组合。
+ **社区支持**：Redux有一个活跃的社区，提供了大量的中间件、工具和丰富的文档。



#### 总结
Redux的设计思路是为了提供一个清晰、可预测和可调试的状态管理解决方案。通过将状态变化逻辑集中管理，Redux使得开发者能够更容易地追踪和维护应用程序的状态。虽然Redux引入了一些额外的概念和步骤，但它的这些原则和模式对于构建大型、复杂的前端应用程序非常有用。
