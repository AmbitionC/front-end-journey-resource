| 特性 | 受控组件 | 非受控组件 |
| :---: | --- | --- |
| **数据管理** | 由 React 组件的 state 管理输入数据 | 由 DOM 元素本身管理数据 |
| **状态更新** | 通过设置 state 来更新 | 通过引用 DOM 元素来更新（通常是 ref） |
| **使用场景** | 当需要实时获取或验证用户输入时，或者当输入控件的值需要由其他 UI 控件控制时 | 当你需要在表单提交时一次性获取表单数据，而不需要控制输入过程中每个变化时 |
| **优点** | 可以轻松地将表单数据集成到组件的 state 中，方便管理和访问 | 较少的代码，更少的性能消耗，因为不需要通过 React 的 state 管理状态 |
| **缺点** | 需要写更多的代码来管理状态和实现双向数据绑定 | 难以实现组件的即时验证、过滤或条件式禁用按钮等交互功能 |


在React中，组件可以根据它们的状态（state）管理方式被分为受控组件（Controlled Components）和非受控组件（Uncontrolled Components）。

#### 受控组件（Controlled Components）
受控组件是指其值（value）由React的状态（state）控制的输入字段。在受控组件中，React的状态是“单一数据源”，这意味着组件的值是由React的状态决定的，而不是由浏览器的DOM决定的。受控组件通常通过以下方式创建：

+ **创建状态**：在组件的构造函数或`useState` Hook中创建一个状态变量。
+ **处理变更**：使用事件处理函数（如`onChange`）来更新状态。
+ **绑定值**：将状态变量绑定到输入字段的`value`属性上。

```jsx
function ControlledInput() {
  const [value, setValue] = useState('');

  function handleChange(event) {
    setValue(event.target.value);
  }

  return (
    <input type="text" value={value} onChange={handleChange} />
  );
}
```



#### 非受控组件（Uncontrolled Components）
非受控组件是指其值不受React状态控制的输入字段。在非受控组件中，输入字段的值由浏览器的DOM直接管理。当用户与输入字段交互时，其值会直接反映在DOM上，而不是存储在React的状态中。非受控组件的使用更接近传统的HTML表单处理方式。

```jsx
function UncontrolledInput() {
  return <input type="text" />;
}
```



#### 受控组件与非受控组件的比较
**（1）受控组件的优点：**

+ 状态一致性：React可以完全控制组件的状态，这使得状态管理更加一致和可预测。
+ 表单验证：受控组件使得在表单提交前进行验证和处理错误更加容易。
+ 性能优化：通过条件渲染和性能优化策略，受控组件可以减少不必要的渲染。

**（2）受控组件的缺点：**

+ 更多的代码：需要额外的事件处理和状态管理代码。
+ 复杂性：对于简单的表单和输入，受控组件可能会增加不必要的复杂性。

**（3）非受控组件的优点：**

+ 简单性：对于简单的用例，非受控组件的代码更简洁。
+ 自然行为：非受控组件的行为更接近传统的HTML表单。

**（4）非受控组件的缺点**：

+ 状态管理：状态分散在DOM中，可能导致状态管理不一致。
+ 验证和处理：需要额外的逻辑来处理表单提交和验证。
