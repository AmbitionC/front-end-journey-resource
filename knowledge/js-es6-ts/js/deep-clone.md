在JavaScript中，深拷贝（deep copy）和浅拷贝（shallow copy）是两种不同的对象复制方式，它们在处理对象、数组或其他复合数据类型时的复制行为上有所区别。

#### 浅拷贝（Shallow Copy）
浅拷贝创建一个新对象，但是这个新对象的属性指向的是原始对象属性所指向的同一个内存地址。换句话说，浅拷贝仅仅复制了最外层的对象或数组，内部的引用类型属性并没有被复制，仍然是指向原始对象中的引用。

```javascript
const original = { a: 1, b: [2, 3] };
const shallowCopy = Object.assign({}, original);
```

在这个例子中，`shallowCopy` 是 `original` 的浅拷贝。`a` 属性的值被复制了，但是 `b` 属性实际上仍然是指向原始数组的引用。因此，如果你修改了 `shallowCopy.b` 的内容，原始数组也会受到影响。



#### 深拷贝（Deep Copy）
深拷贝创建一个新对象，并且递归地复制原始对象中的所有属性。这意味着对象中的所有引用类型属性也会被递归地复制，新对象将不与原始对象共享任何引用。

```javascript
const original = { a: 1, b: [2, 3] };
const deepCopy = JSON.parse(JSON.stringify(original));
```

在这个例子中，`deepCopy` 是 `original` 的深拷贝。`a` 属性的值和 `b` 属性中的数组都被独立地复制了。因此，即使你修改了 `deepCopy.b` 的内容，原始数组也不会受到影响。



#### 深拷贝和浅拷贝的区别
+ **引用类型属性**：浅拷贝只复制了引用类型的引用，而深拷贝复制了引用类型本身及其所有子属性。
+ **内存地址**：浅拷贝的对象和原始对象的引用类型属性可能指向相同的内存地址，而深拷贝的对象和原始对象的引用类型属性指向不同的内存地址。
+ **性能**：深拷贝通常比浅拷贝更耗时，因为它需要递归地复制所有属性。
+ **适用场景**：如果你需要完全独立的副本，不希望原始对象和副本之间有任何关联，那么应该使用深拷贝。如果你只需要复制对象的顶层结构，而不担心内部的引用类型属性，那么浅拷贝可能更合适。
