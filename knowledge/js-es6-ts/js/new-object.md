下表总结了`new` 和 `Object.create` 的主要区别：

| 特性 | `new` | `Object.create` |
| :---: | --- | --- |
| 用法 | 使用构造函数创建新对象 | 使用现有对象作为原型创建新对象 |
| 返回值 | 返回一个由构造函数创建的新对象 | 返回一个新对象，其原型由指定对象确定 |
| 参数 | 构造函数可以接受参数，传递给构造函数 | 第一个参数是要用作新对象原型的对象 |
| 构造函数内部的 `this` | 构造函数内部的 `this`关键字指向新创建的对象 | 构造函数内部没有 `this`关键字，使用原型链来确定属性和方法的值 |
| 实例属性和方法 | 构造函数可以在实例上定义属性和方法 | 实例属性和方法通过原型链继承自原型对象 |
| 兼容性 | 兼容性良好 | 对于一些旧版本浏览器，可能不支持，需要做额外的兼容处理 |


综上所述，`new` 用于使用构造函数创建新对象，并且构造函数内部可以定义实例属性和方法，而 `Object.create` 用于创建一个新对象，其原型由指定对象确定，不涉及构造函数和构造函数内部的 `this`，因此也不支持传递参数。
