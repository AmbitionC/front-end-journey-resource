在JavaScript中，作用域（Scope）和作用域链（Scope Chain）以及执行上下文（Execution Context）是理解和管理变量访问和函数行为的关键概念。

#### 作用域（Scope）
作用域是指一个变量可以被访问的区域。在JavaScript中，作用域主要有两种类型：

**（1）全局作用域**：在代码的最外层声明的变量，这些变量在整个程序中都是可见的。

**（2）局部作用域**：在函数或代码块内部声明的变量，这些变量只能在其声明的函数或代码块内部访问。

JavaScript还支持`let`和`const`声明的块级作用域，它们只在声明它们的代码块（例如`if`语句或`for`循环）中可见。



#### 作用域链（Scope Chain）
作用域链是JavaScript查找变量和函数声明的一系列作用域。当代码在执行上下文中运行时，JavaScript引擎会通过作用域链来确定变量和函数的访问权限。

作用域链的创建遵循以下规则：

+ 全局作用域是作用域链的底部。
+ 当一个函数被调用时，它会创建一个新的作用域，并被添加到作用域链的顶部。
+ 每个函数的作用域链都包含其父函数的作用域，以及全局作用域。
+ 当查找变量或函数时，JavaScript引擎会从作用域链的顶部开始，向下逐层查找，直到找到匹配的声明。



#### 执行上下文（Execution Context）
执行上下文是JavaScript在代码执行时的一个环境，它包含了作用域链、变量对象（VO）和函数信息。每当JavaScript引擎执行代码时，它都会创建一个新的执行上下文。

执行上下文的创建遵循以下规则：

+ 当全局代码执行时，会创建一个全局执行上下文，它的作用域链只包含全局作用域。
+ 当一个函数被调用时，会创建一个新的函数执行上下文，并将其压入执行上下文栈的顶部。
+ 每个执行上下文都有自己的作用域链和变量对象。
+ 执行上下文栈（Execution Context Stack）用于管理执行上下文的生命周期，当一个函数执行完毕后，其执行上下文会被弹出栈。



#### 闭包（Closure）
闭包是一个函数，它可以访问其创建时的作用域链中的变量，即使这个函数在其原始作用域之外被调用。闭包的存在是因为JavaScript的函数可以访问其外部函数的作用域，即使外部函数已经执行完毕。



#### 注意事项
+ 理解作用域和作用域链对于避免变量冲突和正确管理变量生命周期非常重要。
+ 使用`let`和`const`可以创建块级作用域，这有助于避免全局变量的污染。
+ 闭包可以在函数外部访问函数内部的变量，这在创建模块和封装私有数据时非常有用。
+ 执行上下文栈的管理是自动的，但了解其工作原理有助于理解函数调用和返回的行为。
