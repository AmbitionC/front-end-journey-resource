`Set`和`Map`是JavaScript中的两种集合类型，它们都是ES6中引入的新特性，用于存储和操作数据集合。尽管它们都是基于键值对的集合，但它们有一些关键的区别和不同的使用场景。

#### Set（集合）
**（1）特点：**

+ `Set`是一个包含不重复元素的集合。
+ 它只存储唯一的值（即每个值只能出现一次）。
+ `Set`对象提供`add`、`delete`、`has`和`clear`等方法来操作集合。

**（2）使用场景：**

+ 当你需要确保某个值只出现一次时。
+ 用于去除数组中的重复元素。
+ 实现一些需要唯一性的数据结构，如标志列表。

**（3）示例**

```javascript
const mySet = new Set();
mySet.add(1);
mySet.add(2);
mySet.add(2); // 不会被添加，因为2已经存在
console.log(mySet.size); // 输出: 2
```



#### Map（映射）
**（1）特点：**

+ `Map`是一个键值对的集合，其中的键和值可以是任何数据类型。
+ 它提供了`set`、`get`、`has`和`delete`等方法来操作键值对。
+ `Map`对象保持键的插入顺序，这意味着你可以按照添加的顺序迭代键值对。

**（2）使用场景：**

+ 当你需要存储键值对且键是唯一的。
+ 用于创建简单的缓存或字典。
+ 需要保持插入顺序的数据结构。

**（3）示例**

```javascript
const myMap = new Map();
myMap.set('key1', 'value1');
myMap.set('key2', 'value2');
console.log(myMap.get('key1')); // 输出: value1
console.log(myMap.has('key2')); // 输出: true
```



#### 区别总结
+ **数据结构**：`Set`是一个值的集合，而`Map`是键值对的集合。
+ **唯一性**：`Set`中的值是唯一的，`Map`中的键是唯一的。
+ **迭代**：`Set`可以按照添加的顺序进行迭代，`Map`也可以按照键值对的插入顺序进行迭代。
+ **方法**：`Set`提供的方法主要针对集合值的操作，而`Map`提供的方法针对键值对的操作。
