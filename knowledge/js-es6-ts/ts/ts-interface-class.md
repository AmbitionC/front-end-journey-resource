在TypeScript中，接口（Interfaces）和类型别名（Type Aliases）都是定义对象结构的工具，但它们之间存在一些关键的区别和用途：

#### 接口（Interfaces）
+ **定义共享结构**：接口主要用于定义对象的结构，可以被类（通过`implements`关键字）实现，也可以被其他接口扩展。
+ **类继承**：接口可以被类实现，这意味着类将继承接口的所有成员。
+ **结构类型**：接口定义的是结构类型，即它关注对象的“形状”或结构。
+ **可扩展性**：接口可以继承其他接口，形成接口的层级结构。
+ **合并**：接口可以被合并，即多个接口可以被合并为一个接口。
+ **声明合并**：在同一个命名空间中，具有相同名称的不同接口将被合并为一个接口。

```typescript
interface Name {
  first: string;
}

interface Person {
  age: number;
}

interface FullName extends Name, Person {
  last: string;
}
```



#### 类型别名（Type Aliases）
+ **定义类型**：类型别名用于给一个类型命名，这个类型可以是原始类型、联合类型、元组、字面量类型等。
+ **类型重用**：类型别名允许你为复杂的类型定义一个名称，以便在多处重用。
+ **灵活性**：类型别名可以表示任何类型，包括基本类型、联合类型、元组、字面量类型等。
+ **不可扩展**：类型别名不支持扩展或合并。
+ **声明合并**：类型别名不支持声明合并，具有相同名称的多个类型别名将导致错误。

```typescript
type Name = {
  first: string;
};

type Person = {
  age: number;
};

// 下面的代码将导致错误，因为类型别名不支持扩展
// type FullName = Name & Person & { last: string };
```



#### 主要区别
+ **使用目的**：接口用于定义对象的结构，而类型别名用于命名任何类型的类型。
+ **继承和实现**：接口可以被类实现，也可以被其他接口扩展；类型别名不支持这些特性。
+ **合并**：接口可以被合并，类型别名不可以。
+ **重用**：类型别名非常适合重用复杂的类型定义。
