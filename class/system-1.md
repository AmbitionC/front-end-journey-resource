## 概述

计算机系统与网络是每一位前端工程师必须掌握的基础知识。尽管前端开发主要关注浏览器、UI、交互和前端框架，但如果没有对计算机系统和网络原理的理解，你很难深入理解前端性能优化、数据传输原理、浏览器渲染机制、缓存策略以及安全防护等问题。

这门课程将从 **计算机系统组成、硬件、操作系统、进程与线程、内存管理** 到 **计算机网络基础、协议与安全**，系统讲解前端工程师必须掌握的底层知识。通过学习，你能够理解：

1. 浏览器是如何与操作系统和硬件交互的。
2. 前端性能优化背后的系统原理。
3. 网络请求、缓存机制、HTTP/HTTPS 协议原理。
4. 安全问题如 XSS、CSRF 背后的网络和系统基础。


## 为什么要学习计算机系统与网络

很多前端工程师在入行初期，只关注框架和工具（如 React、Vue、Webpack 等），但在工作中会遇到一些系统性问题，例如：

* 页面加载缓慢，但无法准确判断是网络问题还是浏览器渲染瓶颈。
* 数据传输效率低，缓存策略不合理。
* 跨域请求频繁报错，不清楚原因。
* 在处理大型项目时，无法合理利用多线程或异步机制。

这些问题的根源都与 **计算机系统和网络原理** 密切相关。掌握底层原理不仅能解决问题，还能提升你的 **工程思维** 和 **技术深度**，为后续学习性能优化、分布式系统和全栈开发打下坚实基础。

## 计算机系统的组成

一个完整的计算机系统主要由 **硬件** 和 **软件** 两大部分构成。

### 硬件部分

硬件是计算机系统的物理基础，包括：

1. **中央处理器（CPU）**
2. **存储设备（内存、硬盘）**
3. **输入输出设备（键盘、显示器、网络接口等）**

### 软件部分

软件是计算机硬件运行的灵魂，主要包括：

1. **操作系统（OS）**：管理硬件资源，提供程序运行环境。
2. **应用程序**：用户直接使用的软件，如浏览器、IDE、办公软件等。
3. **驱动程序**：连接操作系统与硬件的中间桥梁。

## 计算机系统的工作原理

计算机系统的工作原理可以用“**输入 → 处理 → 输出**”来概括：

1. **输入**：用户或外部设备向计算机发送数据。
2. **处理**：CPU 根据操作系统和程序逻辑对数据进行处理。
3. **存储**：处理结果可能临时存储在内存或永久存储在硬盘。
4. **输出**：处理结果返回给用户或其他设备。

前端开发中，这一过程可以对应为：

* **输入**：用户点击按钮或输入数据。
* **处理**：浏览器解析 JS、执行渲染逻辑。
* **输出**：DOM 更新，页面显示效果。
* **存储**：本地缓存（localStorage、IndexedDB）或后端数据库。

## 计算机硬件基础

### 硬件架构

硬件架构可以分为以下几类：

1. **冯·诺依曼架构**：主流计算机架构，特点是程序和数据存储在同一内存中。
2. **CPU + 内存 + IO 总线架构**：CPU 通过总线访问内存和外设，前端开发中理解总线和缓存能帮助优化性能。

### 中央处理器 CPU

CPU 是计算机的核心，负责指令执行和运算。主要组成：

* **运算器（ALU）**：执行算术和逻辑运算。
* **控制器（CU）**：解析指令并控制数据流向。
* **寄存器**：高速存储器，用于临时存储数据。

前端相关知识点：

* JS 是单线程的，但浏览器通过 **事件循环（Event Loop）** 和 **Web Worker** 利用 CPU 的多线程特性实现异步处理。
* CPU 缓存（L1、L2、L3）影响前端性能优化，例如循环和 DOM 操作的效率。

### 存储设备

存储设备可分为：

1. **内存（RAM）**：高速、易失性，用于存放运行中的程序和数据。
2. **硬盘（HDD/SSD）**：低速、非易失性，用于长期数据存储。
3. **缓存（Cache）**：介于 CPU 与内存之间，提高数据访问速度。

前端性能优化涉及：

* 减少内存泄漏：长期持有 DOM 引用会导致浏览器占用大量内存。
* 利用浏览器缓存：例如 service worker、HTTP 缓存。

### 输入输出设备

输入输出设备是计算机与外界交互的接口，包括：

* **输入设备**：键盘、鼠标、触屏。
* **输出设备**：显示器、打印机、音响。
* **网络接口**：以太网、Wi-Fi 等。

前端开发中，IO 设备主要体现在**浏览器渲染、网络请求和事件响应**上。

## 操作系统

操作系统是计算机硬件和应用程序之间的桥梁，主要作用包括：

1. **资源管理**：CPU、内存、磁盘、IO 设备。
2. **任务调度**：管理进程和线程的执行顺序。
3. **文件系统管理**：提供统一接口访问文件和目录。
4. **安全管理**：权限控制、用户隔离。

### 常见的操作系统

* **桌面系统**：Windows、macOS、Linux。
* **移动系统**：Android、iOS。
* **服务器系统**：大多数采用 Linux。

### 进程与线程

* **进程**：操作系统分配资源的基本单位，每个进程有独立内存空间。
* **线程**：进程的子任务，共享进程资源，轻量级。

前端相关：

* JS 在浏览器中单线程运行，但通过 **异步任务队列、事件循环和 Web Worker** 实现“多线程”效果。
* Node.js 使用事件驱动 + 异步 IO 模型，高效利用 CPU。

### 内存管理

操作系统通过\*\*堆（Heap）**和**栈（Stack）\*\*管理内存：

* **栈**：存放局部变量，先进后出。
* **堆**：存放动态分配对象，垃圾回收机制负责释放内存。

前端中，理解 JS 的内存管理有助于：

* 避免内存泄漏（闭包、事件监听未解绑等）。
* 优化大规模数据处理和渲染性能。

## 计算机网络

### 网络基础

计算机网络是计算机之间交换信息的系统，前端开发主要关注 **互联网和局域网**。

* **IP 地址**：唯一标识网络设备。
* **MAC 地址**：唯一标识网络接口。
* **域名（DNS）**：将域名映射为 IP 地址，方便访问网站。

网络结构：

* **客户端-服务器（C/S）模式**：浏览器作为客户端，服务器提供数据。
* **分布式架构**：大型网站通常使用负载均衡、多节点服务器。

### 网络协议

协议是网络通信的规则。前端常用协议：

1. **HTTP/HTTPS**

   * HTTP 是无状态协议，HTTPS 在 HTTP 上增加了 SSL/TLS 加密。
   * 理解请求方法（GET、POST、PUT、DELETE）、状态码（200、301、404、500）对前端开发非常重要。
2. **TCP/IP**

   * TCP：可靠传输，保证数据完整性。
   * IP：负责数据包寻址和路由。
3. **WebSocket**

   * 建立持久连接，实现实时通信，例如聊天应用或股票行情更新。

### 网络安全

前端需要关注的网络安全问题：

1. **XSS（跨站脚本攻击）**：恶意脚本注入，危害用户数据。
2. **CSRF（跨站请求伪造）**：攻击者利用用户身份发起恶意请求。
3. **HTTPS 加密**：保证数据传输安全。
4. **CORS（跨域资源共享）**：浏览器安全机制，防止非法跨域请求。
