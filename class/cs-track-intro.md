# 计算机基础速通·导读:AI 时代为什么更要打牢底座

很多人以为进入 AI/Agent 时代,计算机基础(网络、操作系统、数据库、缓存)就不重要了——反正模型会写代码。事实恰恰相反:**当你要把一个 Agent 应用做到「能上线、扛得住、不出事」时,决定上限的往往不是 Prompt,而是这些底层功底。**

这条速通路线,就是把求职高频、又最能转化成工程能力的四块基础,按「够用且打通」的顺序串起来。

## 为什么这四块最值得先学

| 模块 | 面试出现频率 | 在 Agent 工程里的真实用途 |
| --- | --- | --- |
| 计算机网络 | 极高 | 流式响应(SSE)、超时重试、HTTPS、长连接、跨域 |
| 操作系统 | 高 | 并发调用模型/工具、进程线程协程、内存与调度 |
| 数据库 / MySQL | 高 | 会话与消息存储、事务一致性、索引与慢查询 |
| Redis / 缓存 | 高 | 应答缓存、限流、分布式锁、热点数据 |

> 一句话:**网络决定你的请求能不能稳定到达,操作系统决定你能并发多少,数据库决定数据对不对,缓存决定快不快、扛不扛得住。**

## 建议的学习顺序

1. **先网络**:从「在浏览器输入 URL 到页面呈现」拉通全局,再深入 TCP 三次握手/四次挥手、TCP vs UDP、HTTP/HTTPS、HTTP 缓存与状态码。对应 **知识库 → 计算机网络** 分组。
2. **再操作系统**:进程/线程/协程的区别 → 同步与互斥 → 死锁 → 虚拟内存与分页 → 进程调度。对应 **知识库 → 计算机基础 → 操作系统**。
3. **接数据库**:索引为什么用 B+ 树 → 索引优化与失效 → 事务 ACID → 隔离级别与 MVCC → 锁 → 日志(redo/undo/binlog)。对应 **知识库 → 计算机基础 → 数据库与 MySQL**。
4. **最后 Redis**:数据结构与场景 → 持久化 → 缓存穿透/击穿/雪崩 → 分布式锁 → 过期与淘汰策略。对应 **知识库 → 计算机基础 → Redis 与缓存**。

## 怎么学才不白学

- **带着问题读**:每篇文章标题本身就是一道高频面试题,先尝试自己答一遍,再对照。
- **用「考一考」自测**:在知识库文章右侧的 Iris 里点「考一考」,让它从本篇随机抽题考你,比反复看更有效。
- **落到工程**:每学一个点,想一想「它在我的 Agent 项目里对应什么」——这正是把八股变成简历亮点的关键。
