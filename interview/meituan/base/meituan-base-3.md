### 面试中的问题：

#### （1）个人背景和学习经历

**答题思路**：简明介绍专业、核心课程、绩点/奖学金等能体现学习能力的亮点，再落到**学习方法**（如何自驱、如何攻克难点）。与技术岗相关的部分多着墨，无关信息一笔带过。

#### （2）课外活动和自我提升

**答题思路**：讲你如何利用课余时间自我提升——做项目、刷题、开源、写博客、参加比赛等。重点体现**主动性和持续投入**，最好能与应聘岗位的技术方向挂钩。

#### （3）前端学习之路

**答题思路**：讲清入门契机、学习路径（HTML/CSS/JS 基础 → 框架 → 工程化）以及过程中遇到的困难和收获。突出**知其所以然**和动手实践的态度，可用一两个具体项目佐证。

#### （4）项目经验分享

**答题思路**：用 STAR 结构挑一个项目深入讲：项目背景、你负责的模块、遇到的**难点与解决方案**、可量化的亮点（性能、体验、复用性）。准备好被追问细节，确保每个技术点都能讲透。

#### （5）手写快速排序算法

**答题思路**：快排基于**分治 + 基准划分**——选一个 pivot，把小于它的放左边、大于的放右边，再递归处理两侧。平均时间复杂度 **O(n log n)**，最坏（已有序且固定取端点）退化为 O(n²)，空间复杂度 O(log n)。

```javascript
function quickSort(arr) {
  if (arr.length <= 1) return arr;
  const pivot = arr[arr.length - 1];
  const left = [], right = [];
  for (let i = 0; i < arr.length - 1; i++) {
    (arr[i] < pivot ? left : right).push(arr[i]);
  }
  return [...quickSort(left), pivot, ...quickSort(right)];
}
```

#### （6）Vue的diff算法

Vue 的 diff 在**同层比较**（不跨层）、**同标签才复用**的前提下，用 **key** 判断节点是否为同一节点。

+ **双端 diff（Vue2）**：新旧子节点各设头尾两个指针，按「旧头-新头、旧尾-新尾、旧头-新尾、旧尾-新头」四种情况两两比较，命中则复用并移动指针，都不命中再用 key 建映射查找，能高效处理头尾增删和乱序。
+ **key 的作用**：帮助复用节点、避免就地复用导致的状态错乱；因此列表渲染不建议用 index 作 key。

#### （7）浏览器的导航过程

从输入 URL 到页面显示：

**（1）URL 解析**与缓存检查 → **（2）DNS 解析**拿到 IP → **（3）建立 TCP 连接**（三次握手，HTTPS 再加 TLS 握手）→ **（4）发送 HTTP 请求** → **（5）服务器响应**返回 HTML → **（6）浏览器解析渲染**（构建 DOM/CSSOM、合成渲染树、布局、绘制、合成）→ **（7）断开或复用连接**。

#### （8）TCP连接和历史连接的优化

**答题思路**：TCP 建连需三次握手是为了**双向确认收发能力并同步初始序列号**，两次无法确认客户端的接收能力、且可能因历史重复报文误建连接。优化手段可讲 **Keep-Alive 长连接复用**（避免重复握手）、连接池，以及 **TLS 会话复用**（Session ID / Session Ticket）减少握手开销。

#### （9）页面渲染过程

浏览器渲染主线：

+ **构建 DOM 树**：解析 HTML。
+ **构建 CSSOM**：解析 CSS，计算样式。
+ **生成渲染树（Render Tree）**：合并 DOM 与样式，剔除不可见节点。
+ **布局（Layout/Reflow）**：计算每个节点的几何位置。
+ **绘制（Paint）**与**分层合成（Composite）**：分图层绘制后由合成线程合成上屏。

注意 **重排（reflow）比重绘（repaint）代价大**，应尽量减少触发。

#### （10）数据结构的探讨

+ **红黑树**：自平衡二叉搜索树，插入/查找/删除均 O(log n)，靠颜色规则近似平衡，常用于内存中的有序结构（如 Java TreeMap）。
+ **B+ 树**：多路平衡树，**数据只存叶子节点且叶子间用链表相连**，树矮胖、一个节点存多个键，减少磁盘 I/O，适合**范围查询**——因此被数据库/文件系统用作索引。

#### （11）操作系统的死锁问题

**死锁**是多个进程互相等待对方持有的资源而都无法推进。

+ **四个必要条件**：互斥、占有并等待、不可剥夺、循环等待。
+ **处理策略**：预防（破坏某一条件，如按序申请资源）、避免（银行家算法）、检测与恢复。
+ **例子**：进程 A 持锁 1 求锁 2，进程 B 持锁 2 求锁 1，形成循环等待。
