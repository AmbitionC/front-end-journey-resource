### 面试问题记录：

#### （1）Java的JVM内存区域

JVM 运行时数据区分为：

+ **程序计数器**：记录当前线程执行字节码的行号，线程私有。
+ **Java 虚拟机栈**：线程私有，存放栈帧（局部变量表、操作数栈）。
+ **本地方法栈**：为 native 方法服务。
+ **堆（Heap）**：线程共享，存放对象实例，是 GC 的主要区域。
+ **方法区**：线程共享，存类元信息、常量、静态变量（JDK8 后由元空间 Metaspace 实现）。

#### （2）静态变量在哪个区域？

静态变量属于**类级别**，存放在**方法区**（JDK8 之后逻辑上归属方法区，实际存于**元空间/堆**中）。所有实例共享同一份静态变量，随类加载而初始化。

#### （3）String a = "123"; String b = new String("123"); a == b

结果为 **`false`**。

+ `a = "123"` 是字面量，指向**字符串常量池**中的对象。
+ `b = new String("123")` 用 `new` 在**堆**上创建了新对象，地址与常量池中的不同。
+ `==` 比较的是**引用地址**，故为 `false`；若比较内容应用 `a.equals(b)`，结果为 `true`。

#### （4）垃圾回收流程

判断对象存活用**可达性分析**（从 GC Roots 出发不可达即为垃圾）。回收算法：

+ **标记-清除**：标记垃圾后清除，会产生内存碎片。
+ **标记-整理**：清除后移动存活对象消除碎片，用于老年代。
+ **复制算法**：把存活对象复制到另一半空间，用于新生代（Eden + 两个 Survivor）。

现代 JVM 采用**分代收集**，配合 G1、CMS 等收集器。

#### （5）面向对象的三大特性

+ **封装（Encapsulation）**：隐藏内部实现，通过访问修饰符只暴露必要接口，降低耦合。
+ **继承（Inheritance）**：子类复用并扩展父类的属性和方法。
+ **多态（Polymorphism）**：同一消息在不同对象上有不同表现，靠**方法重写 + 向上转型 + 动态绑定**实现，提升扩展性。

#### （6）Synchronize用在静态方法和实例方法上的区别

+ **静态方法**：锁的是**类的 Class 对象**（类锁），所有实例共用一把锁，同一时刻只有一个线程能进入任一被锁静态方法。
+ **实例方法**：锁的是**当前实例对象（this）**，不同实例互不影响。

因此类锁与对象锁是两把不同的锁，两个线程分别调静态锁方法和实例锁方法不会互斥。

#### （7）TCP、HTTP、IP协议层

按 TCP/IP 分层：

+ **HTTP**：**应用层**协议，用于网页与应用数据交换。
+ **TCP**：**传输层**协议，提供面向连接、可靠有序的字节流传输。
+ **IP**：**网络层**协议，负责寻址和把数据包在网络间路由转发。

上层协议依赖下层：HTTP 数据交给 TCP 分段，再由 IP 封装成包投递。

#### （8）TCP如何保证可靠性

+ **序列号 + 确认应答（ACK）**：数据按序编号，接收方确认，保证有序不丢。
+ **超时重传**：未收到 ACK 则重发。
+ **校验和**：检测数据是否损坏。
+ **流量控制**：滑动窗口，防止发送过快压垮接收方。
+ **拥塞控制**：慢启动、拥塞避免等，防止网络过载。

#### （9）拥塞避免机制

TCP 拥塞控制的四个核心算法：

+ **慢启动**：拥塞窗口 cwnd 从小指数增长，快速探测可用带宽。
+ **拥塞避免**：cwnd 达到阈值后改为线性增长，谨慎试探。
+ **快重传**：收到 3 个重复 ACK 立即重传丢失报文，不等超时。
+ **快恢复**：快重传后将阈值减半并从该值继续线性增长，而非回到慢启动。

#### （10）HTTPS和HTTP的区别

HTTPS 在 HTTP 之下、TCP 之上加了 **TLS/SSL** 层，提供**加密、身份认证和完整性校验**；HTTP 明文传输不安全。端口上 HTTP 用 80、HTTPS 用 443；HTTPS 需 CA 证书并多一次 TLS 握手。

#### （11）通信加密方式

HTTPS 采用**混合加密**：

+ **非对称加密**（RSA/ECDHE）用于握手阶段**安全协商出对称密钥**，解决密钥分发问题。
+ **对称加密**（AES 等）用协商出的密钥加密后续业务数据，速度快。

兼顾了非对称加密的安全性和对称加密的高效率；同时用证书做身份认证、用消息摘要保证完整性。

#### （12）爬楼梯算法题

**答题思路**：每次可走 1 或 2 级，到第 n 级的走法 `f(n) = f(n-1) + f(n-2)`（本质是斐波那契）。递归会有大量重复计算，用 dp 或滚动变量优化到 **O(n) 时间、O(1) 空间**。

```javascript
function climbStairs(n) {
  if (n <= 2) return n;
  let a = 1, b = 2;
  for (let i = 3; i <= n; i++) {
    [a, b] = [b, a + b];
  }
  return b;
}
```
