**答题思路**：体现「成体系 + 能落地」。可讲：官方文档和经典书打基础 → 视频/博客补广度 → **做项目和造轮子（手写 Promise、简易响应式）把原理内化** → 读源码/规范深挖 → 用笔记或博客沉淀复盘。强调你不是只跟教程，而是动手实践加持续总结。

#### （3）进程和线程的区别？

+ **进程**：操作系统**资源分配**的基本单位，有独立的地址空间，进程间相互隔离，通信要靠 IPC（管道、共享内存、消息队列）。开销大，但一个进程崩溃不影响其他进程。
+ **线程**：CPU **调度**的基本单位，是进程内的执行单元，**共享进程的内存和资源**，通信方便但需加锁防竞争。创建和切换开销小。
+ **一句话**：进程是资源容器，线程是执行流；一个进程可含多个线程。
+ **前端例子**：浏览器是多进程架构（每个标签页/渲染进程隔离），渲染进程内又有 GUI 线程、JS 引擎线程、事件触发线程等，其中 **GUI 渲染与 JS 引擎互斥**，所以 JS 长任务会卡住页面。

#### （4）说下这几个你认为哪个开销更大，io、dom操作、本地内存消耗？

一般排序：**网络 I/O > DOM 操作 > 本地内存读写**。

+ **网络 I/O**：跨机器、受带宽/延迟/RTT 影响，通常毫秒到百毫秒级，最慢且不可控，所以要靠缓存、合并请求、CDN 优化。
+ **DOM 操作**：JS 引擎与渲染引擎跨线程交互，尤其**触发回流/重绘**代价高，频繁读写布局属性会强制同步布局。优化靠批量操作、虚拟 DOM diff、`transform` 走合成层。
+ **本地内存消耗**：纯内存读写是纳秒级，最快；但内存占用过高会引发 GC 频繁或 OOM，间接影响性能。
+ 结论：性能优化优先砍网络和 DOM 开销。

#### （5）那你说下懒加载怎么实现的？

懒加载指**资源延迟到需要时才加载**，减少首屏请求、省流量。以图片为例：

+ **原生方案**：`<img loading="lazy">`，浏览器自动在接近视口时加载，简单但可控性差。
+ **IntersectionObserver（推荐）**：先把真实地址存 `data-src`，用观察器监听图片是否进入视口，进入时把 `data-src` 赋给 `src` 触发加载，然后 `unobserve`。相比监听 `scroll` + `getBoundingClientRect`，它不阻塞主线程、无需节流，性能更好。

```javascript
const io = new IntersectionObserver((entries) => {
  entries.forEach((e) => {
    if (e.isIntersecting) {
      e.target.src = e.target.dataset.src;
      io.unobserve(e.target);
    }
  });
});
document.querySelectorAll('img[data-src]').forEach((img) => io.observe(img));
```

同样思路可用于组件、路由和长列表的懒加载。

#### （6）webpack怎么实现路由懒加载？

核心是用 **ES 的动态 `import()`**：webpack 遇到 `import()` 会自动做**代码分割（code splitting）**，把该模块单独打成一个 chunk，只有路由匹配、组件真正被访问时才发请求加载对应 chunk。

```javascript
const routes = [
  {
    path: '/user',
    // 动态 import 返回 Promise，webpack 拆成独立 chunk
    component: () => import(/* webpackChunkName: "user" */ '@/views/User.vue'),
  },
];
```

+ Vue Router / React（配合 `React.lazy` + `Suspense`）都基于这个机制。
+ 好处是**首屏只加载当前路由代码**，其余按需加载，显著减小首包体积、加快首屏。魔法注释 `webpackChunkName` 可自定义 chunk 名便于缓存与分析。

#### （7）你提到了重绘重排，什么区别？哪些会引起重绘哪些引起重排？

+ **重排/回流（Reflow）**：元素的**几何属性或布局**变化，浏览器需重新计算位置和尺寸。触发：改宽高/边距/字体大小、增删可见 DOM、窗口 resize、以及**读取 `offsetTop`/`scrollHeight`/`getComputedStyle` 等会强制同步布局**的属性。开销大。
+ **重绘（Repaint）**：元素**外观**变化但不影响布局，只需重新绘制像素。触发：改 `color`、`background`、`visibility`、`box-shadow` 等。
+ **关系**：**回流一定触发重绘，重绘不一定回流**。
+ **优化**：合并 DOM 修改、用 `class` 批量改样式、缓存布局读取值避免读写交替、动画用 `transform`/`opacity`（走 GPU 合成层，跳过回流重绘）、`will-change` 提前提升图层。

#### （9）双向数据绑定的原理？

双向绑定 = **数据变更驱动视图** + **视图变更驱动数据**，核心是响应式系统。以 Vue 为例：

+ **数据劫持**：Vue 2 用 `Object.defineProperty` 给属性装 getter/setter；Vue 3 用 `Proxy` 代理整个对象（能监听属性增删和数组索引，性能更好）。
+ **依赖收集与派发更新**：读取属性时（getter）收集用到它的 watcher/副作用（依赖收集）；属性变化时（setter）通知这些依赖去更新视图（发布订阅）。
+ **`v-model` 本质**：是 `:value` 绑定 + `@input` 监听的**语法糖**——数据变了刷新输入框，用户输入了通过事件写回数据，从而形成双向。
