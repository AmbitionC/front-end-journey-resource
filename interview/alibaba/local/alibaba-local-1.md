### 面试中遇到的问题：

#### （1）有哪些常见的数据结构

常见数据结构可分为**线性**与**非线性**两类：

+ **线性结构**：数组、链表、栈（后进先出）、队列（先进先出）、哈希表。
+ **非线性结构**：树（二叉树、堆、字典树等）、图。

选型看**访问模式**：随机访问用数组，频繁增删用链表，键值查找用哈希表，层级/优先级关系用树，网状关系用图。

#### （2）数据结构分类

按**逻辑结构**分为**线性结构**（元素一对一，如数组、链表、栈、队列）和**非线性结构**（一对多的树、多对多的图）。

按**存储（物理）结构**分为**顺序存储**（连续内存，如数组）和**链式存储**（靠指针链接，如链表），此外还有散列存储、索引存储。

#### （3）物理结构和逻辑结构

+ **逻辑结构**：数据元素之间的**抽象逻辑关系**，与内存无关，如线性、树形、图状。
+ **物理结构**（存储结构）：逻辑结构在**内存中的实际存放方式**，主要是顺序存储和链式存储。

同一逻辑结构可以有不同物理实现，例如「栈」这个逻辑结构既能用数组（顺序）也能用链表（链式）实现。

#### （4）链表的增删改查时间复杂度 应用场景

+ **增/删**：已知节点位置时只需改指针，`O(1)`；但若需先定位到该位置，则要 `O(n)`。
+ **查/改**：不支持随机访问，只能从头遍历，`O(n)`。

**应用场景**：频繁在中间插入删除、且不需要随机访问的场景，如 LRU 缓存（哈希表 + 双向链表）、浏览器前进后退、大整数运算等。

#### （5）树有哪些常见的 应用场景有哪些

+ **二叉搜索树 / 平衡树（AVL、红黑树）**：有序数据的查找、插入、删除，均摊 `O(log n)`；红黑树用于 Java TreeMap、C++ map。
+ **B 树 / B+ 树**：多路平衡树，减少磁盘 I/O，用于数据库索引、文件系统。
+ **堆**：优先队列、Top K、堆排序。
+ **字典树（Trie）**：前缀匹配、搜索联想、敏感词过滤。

#### （6）堆的应用场景

堆是一棵**完全二叉树**，大顶堆父节点大于子节点，小顶堆相反，能在 `O(1)` 拿到极值、`O(log n)` 插入/删除。

+ **优先队列**：任务调度、Dijkstra 最短路。
+ **Top K 问题**：求最大 K 个用小顶堆、最小 K 个用大顶堆。
+ **堆排序**：原地排序，时间复杂度 `O(n log n)`。

#### （7）图的算法有哪些

+ **遍历**：深度优先（DFS，用栈/递归）、广度优先（BFS，用队列）。
+ **最短路径**：Dijkstra（非负权）、Bellman-Ford（可含负权）、Floyd（多源）。
+ **最小生成树**：Prim、Kruskal。
+ **拓扑排序**：处理有向无环图（DAG）的依赖顺序，如任务调度。

#### （8）动态规划是怎么实现的

**核心**：把大问题拆成**重叠子问题**，用一个表保存子问题结果避免重复计算。实现三步：

**（1）定义状态**：`dp[i]` 表示什么。

**（2）写转移方程**：`dp[i]` 与前面状态的关系。

**（3）确定初始值和遍历顺序**。

自顶向下用**递归 + 记忆化**，自底向上用**迭代填表**。适用于最优子结构且子问题重叠的问题（背包、最长公共子序列等）。

#### （9）计算机网络通信的过程（传输层）

传输层负责**端到端**的进程间通信，靠**端口号**区分应用。主要有 **TCP** 和 **UDP**：

+ **TCP**：面向连接、可靠传输。三次握手建连、四次挥手断连，通过序列号/确认应答、超时重传、滑动窗口、流量控制与拥塞控制保证有序可靠。
+ **UDP**：无连接、不保证可靠，但开销小、延迟低，适合音视频、直播、DNS 查询。

#### （10）计算机网络硬件有哪些

+ **网卡（NIC）**：主机接入网络的接口。
+ **中继器 / 集线器（Hub）**：物理层，信号放大与广播。
+ **交换机（Switch）**：数据链路层，按 MAC 地址转发。
+ **路由器（Router）**：网络层，按 IP 跨网段选路。
+ **调制解调器（Modem）**：数模信号转换。

#### （11）项目有部署上线吗

**答题思路**：这是考察工程完整性。如实说明部署流程即可：代码托管（Git）→ 打包构建（Vite/Webpack）→ 部署方式（静态资源上传 CDN/OSS、Nginx 托管、或 Docker + 云服务器）→ 域名与 HTTPS 配置。若有 CI/CD（GitHub Actions 自动构建部署）可作为亮点强调。

#### （12）介绍一些项目里主要负责什么部分

**答题思路**：先一句话交代项目背景和你的角色，再分点说负责的模块（如某个业务页面、通用组件、状态管理、性能优化）。突出**你独立完成或主导**的部分，并带上量化结果（性能提升多少、复用率等），避免只说「我负责前端」这种笼统回答。

#### （13）主要介绍其中一个部分

**答题思路**：挑一个**有技术含量、你最熟**的模块深入讲。按「需求背景 → 技术方案与选型理由 → 关键实现难点 → 最终效果」组织。重点讲清你的**思考和取舍**，而不是罗列功能。

#### （14）这部分中有没有遇到什么困难 怎么解决的

**答题思路**：用 STAR 结构（情境-任务-行动-结果）。选一个**真实且有一定深度**的问题，讲清你如何定位（排查手段、工具）、尝试了哪些方案、为什么选最终方案、结果如何。体现分析和解决问题的能力，不要说「百度一下就好了」。

#### （15）css的postion有哪些

+ **static**：默认值，正常文档流，`top/left` 无效。
+ **relative**：相对**自身原始位置**偏移，不脱离文档流。
+ **absolute**：脱离文档流，相对**最近的非 static 定位祖先**定位。
+ **fixed**：脱离文档流，相对**视口**定位。
+ **sticky**：粘性定位，在阈值内表现为 relative，超过后表现为 fixed。

#### （16）relative主要是相对于什么

`position: relative` 是相对于**元素自身在正常文档流中的原始位置**进行偏移。它**不脱离文档流**，原来占据的空间仍然保留（不会被其他元素填补），常被用作 `absolute` 子元素的定位参照物（包含块）。

#### （17）盒子模型

每个元素都是一个矩形盒子，从内到外由 **content（内容）→ padding（内边距）→ border（边框）→ margin（外边距）** 组成。

+ **标准盒模型**（`content-box`）：`width` 只算内容宽度。
+ **IE / 怪异盒模型**（`border-box`）：`width` 包含 content + padding + border。

用 `box-sizing` 切换，实际开发常设 `border-box` 便于尺寸计算。

#### （18）css的常见单位

+ **绝对单位**：`px`（像素）。
+ **相对字体单位**：`em`（相对父元素字号）、`rem`（相对根元素 html 字号）。
+ **相对容器单位**：`%`（相对父元素）。
+ **视口单位**：`vw`/`vh`（视口宽高的 1%）、`vmin`/`vmax`。

移动端适配常用 `rem` + 动态根字号，或 `vw` 方案。

#### （19）js原型链

每个对象都有一个内部 `[[Prototype]]`（`__proto__`），指向其构造函数的 `prototype`。访问属性时若自身没有，就沿着 `__proto__` 逐级向上查找，直到 `Object.prototype`（其原型为 `null`），这条链就是**原型链**。

它是 JS **继承**的基础，也是 `instanceof` 的判断依据。查找越深性能越低。

#### （20）基本数据类型

JS 有 7 种基本类型：**Number、String、Boolean、Undefined、Null、Symbol、BigInt**，它们存储在**栈**上、按值传递。

除此之外的都是 **Object**（引用类型，存堆、按引用传递）。注意 `typeof null` 返回 `"object"`，是历史遗留 bug。

#### （21）forEach怎么退出

`forEach` **无法用 `break` 或 `return` 中断**（`return` 只跳过当前次）。若需提前退出：

+ 改用 `for` / `for...of` 循环，用 `break`。
+ 用 `some()`（返回 `true` 时停止）或 `every()`（返回 `false` 时停止）。
+ 抛异常配合 `try/catch`（不推荐，可读性差）。

#### （22）http和https区别

+ **安全性**：HTTP 明文传输；HTTPS 在 HTTP 和 TCP 之间加了 **SSL/TLS** 层，对传输内容加密。
+ **端口**：HTTP 默认 80，HTTPS 默认 443。
+ **证书**：HTTPS 需向 CA 申请证书验证服务端身份。
+ **性能**：HTTPS 有握手和加解密开销，但保证机密性、完整性、身份认证，防窃听和中间人攻击。

#### （23）ca证书有哪些类别有了解过吗

按**验证级别**分为三类：

+ **DV（域名验证）**：只验证域名所有权，签发快、免费的居多（如 Let's Encrypt），适合个人站。
+ **OV（组织验证）**：额外验证企业真实性，适合企业官网。
+ **EV（扩展验证）**：最严格的审核，适合银行、电商等对信任要求高的场景。

证书本身遵循 X.509 标准，由受信任的根 CA 逐级签发形成信任链。

#### （24）有抓包过吗 抓过哪些 用什么抓的

**答题思路**：如实回答用过的工具与场景即可。常见工具：

+ **Chrome DevTools Network 面板**：看请求头/响应、状态码、耗时、缓存。
+ **Fiddler / Charles**：HTTP(S) 抓包、断点改包、弱网模拟、手机代理抓包。
+ **Wireshark**：更底层，能抓 TCP/UDP 甚至握手包，用于分析网络层问题。

可结合真实排查经历说明抓过什么（如接口报错、HTTPS 握手、跨域预检）。
