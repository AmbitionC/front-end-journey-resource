### 面试中的问题：

#### （1）自我介绍

**答题思路**：1 分钟讲清「你是谁 + 技术亮点 + 为什么匹配」。建议结构：教育背景 → 最拿得出手的 1~2 个项目、你的角色与量化产出（性能提升 X%、覆盖多少用户）→ 熟悉的技术栈（Vue/React、工程化）→ 想来该岗位的原因。多用与 JD 相关的关键词，避免记流水账。

#### （2）项目中的重难点

**答题思路**：用 **STAR** 结构讲一个真实难点——背景（Situation）、任务（Task）、你的动作（Action）、结果（Result）。重点讲「为什么难」和「你的技术决策与取舍」，例如首屏慢如何定位（Performance 面板/Lighthouse）、试了哪些方案、最终选型的理由和量化收益。突出你的独立思考，而非罗列技术名词。

#### （3）为什么想学前端？什么时候开始学的

**答题思路**：给一个真实、具体的动机，比一句「感兴趣」更可信。可从「结果即时可见、成就感强」「兼顾工程与设计/交互」「贴近用户」等角度切入，配一个具体触发点（做的第一个页面、某门课/某个项目）。时间点如实说，并顺势带出你持续投入的证据（做过的项目、系统学过什么）。

#### （4）怎么学习前端的

**答题思路**：体现「有方法、成体系、能落地」。可讲：官方文档 + 经典书（《你不知道的 JS》《CSS 世界》）打基础 → 视频/博客补广度 → **通过做项目和造轮子（手写 Promise、简易框架）把知识内化** → 读源码/看规范深挖原理 → 用博客或笔记沉淀。强调你不是只看教程，而是动手 + 复盘。

#### （5）B站上喜欢看哪些前端UP主

**答题思路**：这是轻松题，考察你是否真在持续学习。如实说 1~2 个你确实看过的，并简单点评「从他那学到了什么」（比如某人讲源码透彻、某人讲工程化实战）。别硬编不熟悉的名字，被追问会露怯。看官方文档/英文资料也可以直说。

#### （6）Vue的核心成员知道吗？

作者是**尤雨溪（Evan You）**。核心团队中较知名的还有 **HcySunYang（霍春阳）**——《Vue.js 设计与实现》作者，深度参与 Vue 3 响应式；**Anthony Fu**——VueUse、Vitest、UnoCSS 等工具作者，也是 Vite 生态核心。周边生态还有 Pinia/Vue Router 的维护者 **Eduardo（posva）**。答不全没关系，重点是能说出尤雨溪并了解 Vue 3 的关键贡献者即可。

#### （7）浏览器安全方面的问题

以 **HTTPS/SSL 证书**为例：SSL/TLS 证书由受信任的 CA 签发，作用是**身份认证**（证明你访问的是真站点，防中间人伪装）和**加密传输**。

+ **握手过程**：客户端校验证书合法性（是否可信 CA 签发、是否过期、域名是否匹配）→ 用证书里的公钥协商出对称密钥 → 后续用对称加密通信，兼顾安全与性能。
+ **意义**：防窃听、防篡改、防钓鱼。浏览器对非 HTTPS 站点会标「不安全」。

#### （8）浏览器插件做的请求会不会被黑客拦截，怎么拦截

有可能。风险点主要来自两方面：

+ **传输层**：如果插件走 HTTP 明文请求，处于同一网络的攻击者可通过**中间人攻击（MITM）**、ARP 欺骗、公共 WiFi 抓包来窃听或篡改。走 HTTPS 并校验证书能大幅降低此风险。
+ **插件生态本身**：恶意插件或被劫持的插件可通过 `webRequest`/`declarativeNetRequest` API **拦截、修改浏览器内所有请求**，甚至注入脚本窃取页面数据。所以浏览器用**权限声明 + 沙箱**限制插件能力，用户应只装可信来源插件。防护上：全站 HTTPS、敏感数据端到端加密、最小权限原则。

#### （9）对于前端安全有什么认识

+ **XSS（跨站脚本）**：攻击者注入恶意脚本在受害者浏览器执行，窃取 Cookie/token。防御：对用户输入/输出**转义**，用框架默认的文本插值（Vue `{{}}`、React `{}` 自动转义），避免 `v-html`/`innerHTML`，配置 **CSP** 限制脚本来源，Cookie 加 `HttpOnly`。
+ **CSRF（跨站请求伪造）**：诱导已登录用户在第三方站点发出携带 Cookie 的请求。防御：**CSRF Token**、`SameSite` Cookie、校验 Referer/Origin。
+ **其他**：点击劫持（`X-Frame-Options`）、依赖投毒、敏感信息不落前端。核心原则是**不信任任何客户端输入**。

#### （10）Vue的性能优化有了解过吗？

+ **渲染层**：`v-for` 加稳定 `key`；`v-if` 和 `v-for` 不要同层；用 `v-show` 替代频繁切换的 `v-if`；长列表用虚拟滚动；`computed` 缓存代替模板里复杂计算；合理用 `v-once`/`v-memo`。
+ **组件层**：路由和组件**懒加载**（`() => import()`）+ 代码分割；`keep-alive` 缓存组件；事件/定时器及时销毁防内存泄漏。
+ **响应式层**：不需要响应式的大对象用 `Object.freeze` 或 `shallowRef` 跳过深度代理。
+ **打包层**：Tree-shaking、Gzip/Brotli、CDN、按需引入组件库、开启构建产物分析。

#### （11）v-if和v-show的区别

+ **实现机制**：`v-if` 控制元素的**创建与销毁**（DOM 真正增删，含组件生命周期和事件绑定）；`v-show` 只切换 CSS `display`，元素**始终存在于 DOM**。
+ **开销**：`v-if` 切换成本高、初始渲染成本低（false 时不渲染）；`v-show` 初始必渲染、切换成本低。
+ **选型**：频繁切换用 `v-show`；条件很少改变、或初始可能不渲染用 `v-if`。另外 `v-if` 支持 `v-else`，可配合 `<template>` 做分组条件。

#### （12）浏览器的渲染原理

**关键渲染路径**：解析 HTML 生成 **DOM 树** → 解析 CSS 生成 **CSSOM 树** → 合并为 **Render 树（只含可见节点）** → **Layout/回流**计算几何位置 → **Paint/绘制**填充像素 → **Composite** 图层合成上屏。

+ **回流（Reflow）**：几何属性变化（尺寸、位置、增删节点、读取 `offsetHeight` 等）触发重新布局，**开销大**。
+ **重绘（Repaint）**：仅外观变化（颜色、背景）不影响布局，开销较小。**回流必然引起重绘，反之不然**。
+ **优化**：批量修改 DOM、用 `class` 一次改样式、`transform`/`opacity` 走合成层不触发回流、缓存布局读取值避免强制同步布局。

#### （13）关于渲染的场景题

**答题思路**：先定位瓶颈（首屏慢？滚动卡顿？大量 DOM？），再对症下药。常见方案：

+ **首屏**：SSR/预渲染、路由懒加载、资源压缩与 CDN、骨架屏、关键 CSS 内联。
+ **长列表/大量 DOM**：虚拟滚动只渲染可视区、分片渲染（`requestAnimationFrame`/时间切片）。
+ **频繁重排**：合并 DOM 操作、用 `DocumentFragment`、动画用 `transform`/`opacity`、`will-change` 提升图层。
+ **数据层**：防抖节流、请求合并缓存。回答时先讲思路再举一个具体优化，体现分析能力。

#### （14）接下来最想学习的前端技术

**答题思路**：选一个**与岗位相关、且你能说出所以然**的方向，别泛泛而谈。可结合自身短板或行业趋势，如 Node.js/BFF 全栈能力、前端工程化（Vite/Monorepo）、性能与稳定性监控、跨端（React Native/小程序）、或 AI 应用方向。关键是说清**为什么学它、打算怎么学**，体现规划性而非跟风。

#### （15）为什么想学这个？

**答题思路**：把动机落到**具体问题或价值**上，而非「大家都在学」。好的框架是「我在做 X 时遇到 Y 痛点 / 想补齐 Z 能力，而这项技术正好能解决」。比如学 Node 是为了打通前后端、独立完成需求；学工程化是为了提升团队协作和构建效率。体现你有目标、能自驱。

#### （16）对于你理想中的公司，你对它有哪些方面的要求

**答题思路**：突出**成长和价值实现**，弱化纯待遇诉求。可讲：有技术氛围和代码规范能学到东西、业务有挑战能锻炼能力、团队协作与 code review 文化好、有明确成长路径。最好和应聘公司的特点呼应，表现出你了解并认同它。避免只谈薪资、加班少这类容易减分的点。

#### （17）项目中还有没有其他亮点想说的

**答题思路**：这是主动加分的机会，提前准备 1~2 个前面没讲过的亮点。可选：**性能优化的量化成果**、封装的通用组件/工具被复用、解决过的疑难 bug、推动的工程化改进、或独立负责的模块。用「做了什么 + 带来什么可衡量收益」的方式讲，体现你的主动性和工程思维。

#### （18）相比于其他人，你有什么优势吗？

**答题思路**：挑 2~3 个**有证据支撑**的优势，每个配一句事例，别空喊「学习能力强」。可从三层组织：**技术**（某方向有深入实践/造过轮子/读过源码）、**工程素养**（注重规范、写测试、会性能优化）、**软素质**（沟通协作、抗压、主动补位）。结合 JD 要求挑最匹配的讲，落到具体成果上更有说服力。

#### （19）反问

**答题思路**：反问要体现你对岗位的兴趣和思考。可问：**团队的技术栈和工程化现状**、这个岗位主要负责的业务与挑战、**新人的成长路径和 mentor 机制**、团队的 code review / 技术分享氛围、对候选人半年内的期望。避免一上来就问薪资、调休、加班等敏感细节；也别问一搜就有答案的公司基本信息，显得没做功课。
