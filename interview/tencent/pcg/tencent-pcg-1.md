### 面试中的问题：

#### （1）实习过程中技术含量较高的地方有哪些？

**答题思路**：挑 1~2 个有深度的点（性能优化、复杂逻辑抽象、工具/基建建设），讲清技术挑战、你的方案与产出。用具体技术名词和量化结果体现「含量」，避免泛泛而谈功能。

#### （2）做可视化项目的心得体会是什么？

**答题思路**：讲对可视化的理解（数据到图形的映射、性能、交互体验），遇到的挑战（大数据量渲染用 Canvas/WebGL、图表库选型 ECharts/D3、自适应布局）以及你的收获与沉淀。

#### （3）依赖分析工具项目与市面上同类产品的优势是什么？技术细节是如何实现的？

**答题思路**：讲清工具解决的问题（依赖体积/冗余/版本冲突可视化）与差异化优势，再展开下面几个技术细节。

+ **如何测量单测覆盖率？编写单元测试的方法是什么？**：用 Jest `--coverage` 或 Istanbul/nyc 插桩，统计语句/分支/函数/行覆盖率；单测方法为隔离单元、mock 外部依赖、遵循 AAA（Arrange-Act-Assert）结构、覆盖边界与异常用例。

+ **Web UI 中的具体布局是怎样的？展示了哪些信息？**：属项目细节，讲清如何以依赖树/体积占比/冲突标记等关键信息组织视图，以及交互（搜索、下钻、高亮冲突）的设计思路。

+ **如何判断包在项目中存在版本冲突？如何处理多版本冲突？**：通过解析 lockfile / 依赖树，发现同一包被引入多个版本即标记冲突；处理方式为统一版本（`overrides`/`resolutions`）、`npm dedupe` 去重，或对不兼容的大版本按需隔离共存。

#### （4）在自己开发过程中，如何处理项目中出现的多版本冲突？引申到实习中处理依赖迁移时遇到的问题。

**技术要点**：用 `npm ls <pkg>` 查看依赖树定位多版本来源；通过 lockfile 锁定、`resolutions`（yarn）/`overrides`（npm）强制统一版本、`npm dedupe` 提升去重；打包层用 webpack `resolve.alias` 指向单一副本；确需共存的（如不兼容大版本）保持隔离。

**答题思路**：迁移场景讲清如何评估破坏性变更、灰度替换、回归验证。

#### （5）了解哪些性能优化手段？

+ **加载**：代码分割、路由/组件懒加载、Tree-shaking、资源压缩、CDN、HTTP2、预加载/预连接。
+ **渲染**：减少重排重绘、虚拟列表、防抖节流、Web Worker 分担计算。
+ **缓存**：HTTP 缓存、Service Worker、接口数据缓存。
+ **首屏**：SSR/预渲染、骨架屏。指标用 FCP/LCP/TTI/CLS，靠 Lighthouse / Performance API 量化。

#### （6）常见的应用层协议有哪些？传输层呢？SSE的原理是什么？

+ **应用层**：HTTP/HTTPS、WebSocket、DNS、FTP、SMTP。
+ **传输层**：TCP（可靠、面向连接）、UDP（无连接、低延迟）。
+ **SSE（Server-Sent Events）**：基于 HTTP 的服务器**单向推送**，响应 `Content-Type: text/event-stream` 保持长连接，浏览器用 `EventSource` 接收并自动重连；比 WebSocket 更轻量，但只能服务器→客户端。

#### （7）HTTP2与HTTPS的关系？了解QUIC协议吗？

+ **关系**：二者不同维度——**HTTPS = HTTP + TLS 加密**，**HTTP/2** 是协议版本（多路复用、头部压缩 HPACK、服务器推送、二进制分帧）。浏览器实现 HTTP/2 基本都要求 TLS，故常一起出现，但并不等价。
+ **QUIC**：基于 **UDP** 的传输协议，是 HTTP/3 的底层。解决 TCP 队头阻塞，内置 TLS1.3（0-RTT 握手更快），支持连接迁移（切换网络不断连）。

#### （8）对ESLint与Prettier的应用场景了解吗？在自己的项目中做过哪些配置？

+ **ESLint**：代码**质量/规范**检查（潜在错误、未用变量、风格规则），可自动修复部分问题。
+ **Prettier**：专注代码**格式化**（缩进、引号、换行），不管逻辑。
+ **协作**：用 `eslint-config-prettier` 关闭 ESLint 中与格式冲突的规则，让 Prettier 管格式、ESLint 管质量；再配 husky + lint-staged 在提交前自动校验。

#### （9）心算题：5点15分时针与分针的夹角是多少度？

**答案：67.5°**。时针每小时走 30°、每分钟走 0.5°，5:15 时针位置 = `5×30 + 15×0.5 = 157.5°`；分针 = `15×6 = 90°`；夹角 = `157.5 − 90 = 67.5°`。

#### （10）是否有其他公司offer以及到岗时间？

**答题思路**：如实回答。有 offer 可适当体现被认可（不夸大、不用来施压）；到岗时间给明确区间，展现对该岗位的契合度与诚意。

#### （11）平时有用过AI吗？

**答题思路**：结合实际讲用 AI 提效的场景（代码补全、debug、写单测、查文档），体现你会用工具但有判断力——会核验 AI 输出、理解原理而非盲目依赖。

#### （12）对 GPT 的接口做了哪些拓展，讲了下如何实现上下文历史记忆，又问我这个做法与网络中的开源项目有什么区别

**技术要点**：上下文记忆本质是每次请求把历史对话（`messages` 数组，含 `role: user/assistant`）一起发给模型——模型本身无状态。工程上维护会话历史，超长时做**滑动窗口**截断或**摘要压缩**旧对话，或用**向量检索（RAG）**召回相关历史注入 prompt。

**答题思路（对比开源项目）**：从是否有持久化存储、记忆压缩策略、多轮 token 成本控制、是否结合向量库做长期记忆等维度展开对比。
