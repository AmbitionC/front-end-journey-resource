### 面试中的问题：

#### （1）自我介绍

**答题思路**：1 分钟讲清「你是谁 + 技术亮点 + 为什么匹配」。结构建议：教育背景 → 最拿得出手的 1~2 个项目及量化产出 → 技术栈 → 想来该岗位的原因。多用与 JD 相关的关键词，避免流水账。

#### （2）籍贯

**答题思路**：如实简短回答即可，属于破冰闲聊，无需展开。

#### （3）介绍下本专业

**答题思路**：简述专业方向与核心课程，重点带到与前端/编程相关的部分（如数据结构、计算机网络、操作系统），为后续技术能力做铺垫。

#### （4）为什么转前端

**答题思路**：讲真实契机（如项目中接触、对可视化/交互的兴趣），突出你为此做的准备（自学、项目、实习），体现主动性和对前端的理解，避免贬低原专业。

#### （5）对 http 协议的理解

HTTP 是**应用层协议**，基于 TCP，采用请求-响应模型，**无状态**（靠 Cookie/Session/Token 维持会话状态）。它定义了客户端与服务器通信的报文格式。HTTP/1.1 默认长连接，HTTP/2 支持多路复用与头部压缩。

#### （6）http 包含哪几个部分

+ **请求**：请求行（方法 + URL + 版本）+ 请求头 + 空行 + 请求体。
+ **响应**：状态行（版本 + 状态码 + 短语）+ 响应头 + 空行 + 响应体。

#### （7）请求头有哪些字段，使用场景

+ **User-Agent**：客户端标识。
+ **Accept / Accept-Encoding / Accept-Language**：可接受的类型/压缩/语言（内容协商）。
+ **Content-Type**：请求体类型（`application/json`、`multipart/form-data`）。
+ **Authorization**：认证凭证（Token）。
+ **Cookie**：携带会话信息。
+ **Cache-Control / If-None-Match / If-Modified-Since**：缓存控制与协商缓存。
+ **Host / Origin / Referer**：主机、跨域来源、来源页。

#### （8）说下浏览器缓存，在不同场景有哪些区别

+ **强缓存**：`Cache-Control`（`max-age`）、`Expires`，命中则直接读本地、不发请求（显示 200 from cache）。
+ **协商缓存**：强缓存失效后发请求，带 `If-None-Match`（对应 `ETag`）/`If-Modified-Since`（对应 `Last-Modified`），服务器判断未变则返回 **304**，用本地副本。

优先级：`Cache-Control` > `Expires`，`ETag` > `Last-Modified`。

#### （9）浏览器缓存与浏览器存储有哪些区别与联系，各自适用场景

+ **HTTP 缓存**：浏览器自动管理，目的是减少网络请求，由响应头控制，针对资源。
+ **浏览器存储**：由 JS 主动读写业务数据——`localStorage`（持久、约 5M、同源）、`sessionStorage`（会话级）、`Cookie`（约 4K、随请求发送）、`IndexedDB`（结构化大容量、异步）。

区别在于缓存是「资源自动复用」，存储是「主动存取业务数据」。
