### 面试中的问题：

#### （1）自我介绍

**答题思路**：用 1 分钟讲清「你是谁 + 技术亮点 + 为什么匹配」。建议结构：教育背景 → 最拿得出手的 1~2 个项目及你的角色和量化产出 → 熟悉的技术栈 → 想来该岗位的原因。多用与 JD 相关的关键词，避免流水账。

#### （2）三栏布局如何实现

两侧固定、中间自适应，现代方案首选 Flex 或 Grid：

```css
/* Flex */
.container { display: flex; }
.left, .right { width: 200px; }
.center { flex: 1; }
```

+ **Grid**：`grid-template-columns: 200px 1fr 200px`。
+ **绝对定位**：左右 absolute 定宽，中间用 `margin` 留出两侧宽度。
+ **圣杯/双飞翼**：float + 负 margin 让中间栏优先渲染（老方案，现已少用）。

#### （3）url 输入到渲染

**（1）URL 解析 + DNS**：解析域名，逐级查询（浏览器缓存→系统→路由→DNS 服务器）得到 IP。

**（2）建立连接**：TCP 三次握手，HTTPS 再做 TLS 握手协商密钥。

**（3）请求与响应**：发送 HTTP 请求，服务器返回 HTML。

**（4）解析渲染**：解析 HTML 构建 **DOM**、解析 CSS 构建 **CSSOM**，合成 Render Tree → Layout（回流）→ Paint → Composite；遇到 JS 会阻塞解析（可用 async/defer）。

#### （4）个人技术栈

**答题思路**：分层次讲——熟练（HTML/CSS/JS/ES6+、Vue 或 React 全家桶）、了解（TypeScript、Node、构建工具、小程序）。突出你最深入的一两个并能讲清原理，避免堆砌不熟的名词（会被追问）。

#### （5）说说闭包

闭包是指函数能访问其**定义时所在词法作用域**的变量，即使该函数在外部执行——内部函数持有外部变量的引用即形成闭包。

+ **场景**：数据私有化（模块模式）、函数柯里化、防抖节流、循环中保存变量。
+ **问题**：被引用的变量无法释放，滥用会导致内存泄漏，需及时解除引用。

#### （6）js 数据类型

+ **原始类型（7 种）**：`number`、`string`、`boolean`、`null`、`undefined`、`symbol`、`bigint`，存于栈，按值传递。
+ **引用类型**：`object`（含 array、function、Date 等），存于堆，按引用传递。

注意 `typeof null` 返回 `'object'`（历史遗留 bug），精确判断类型用 `Object.prototype.toString.call()`。

#### （7）Vue 的 diff 实现原理

+ **同层比较**：只对比同层级节点、不跨层，把复杂度控制在 **O(n)**。
+ **key 复用**：通过 key 标识节点身份，快速判断能否复用真实 DOM。
+ **算法**：Vue2 采用**双端比较**（新旧头尾四指针两两比对），Vue3 引入**最长递增子序列**优化节点移动，减少 DOM 操作。

#### （8）index 的作用

在 `v-for` 列表渲染中，`key` 用于 diff 时标识节点身份。用**数组 index 作 key** 存在隐患：列表增删或排序后 index 会变，导致 key 与内容错位，Vue 复用了错误的 DOM，引发状态错乱（如输入框内容错位）和不必要的更新。应使用**稳定唯一的 id** 作为 key。

#### （9）讲一个实习产出

**答题思路**：用 STAR 结构讲一个具体成果——你负责的模块、解决的问题、用的技术方案、以及**量化收益**（如首屏从 3s 降到 1s、复用组件减少 X% 重复代码）。突出你的贡献和思考，而非团队整体。
