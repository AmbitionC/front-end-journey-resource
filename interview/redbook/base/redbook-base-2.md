#### （3）什么是虚拟内存？

虚拟内存是操作系统的内存管理机制：给每个进程一段**连续的虚拟地址空间**，通过**页表**把虚拟页映射到物理内存的页框；物理内存不够时，把暂时不用的页换出到磁盘（swap），用到时再换入。

+ **好处**：进程间内存隔离、可用地址空间大于物理内存、按需分页（Demand Paging）延迟加载。
+ **代价**：缺页中断（Page Fault）需读磁盘，换页过于频繁会引发抖动（Thrashing）。

#### （4）什么是WebSocket，它的原理与实现

**WebSocket** 是建立在 TCP 之上的**全双工**通信协议，允许客户端与服务端在一条长连接上双向实时收发数据。

+ **建连原理**：借助一次 HTTP 请求完成「协议升级」——客户端带 `Upgrade: websocket` 和 `Sec-WebSocket-Key`，服务端返回 `101 Switching Protocols` 与校验后的 `Sec-WebSocket-Accept`，握手完成后复用该 TCP 连接。
+ **数据传输**：以**帧（frame）** 为单位收发，头部开销极小，服务端可主动推送。
+ **实现**：浏览器端 `new WebSocket(url)`，监听 `onopen/onmessage/onclose/onerror`，用 `send()` 发送；服务端可用 `ws`、Socket.IO 等库。适合聊天、实时推送、协同场景。

#### （5）WebSocket与HTTP的区别，以及两者在实现上有什么不一样？优缺点是什么？

+ **通信模式**：HTTP 是**请求-响应**、半双工，服务端无法主动推送；WebSocket 是**全双工**长连接，双向实时。
+ **连接与开销**：HTTP 每次请求都带完整头部、默认短连接（keep-alive 复用也是串行）；WebSocket 一次握手后保持连接，后续帧头部极小。
+ **状态**：HTTP 无状态；WebSocket 建连后保持会话。
+ **优缺点**：WebSocket 实时性强、开销低，但需自行处理心跳保活、断线重连；HTTP 简单、无状态、利于缓存和横向扩展，适合普通请求。**选型**：实时推送用 WebSocket，普通接口用 HTTP。

#### （6）场景题：不借助服务器端，用WebSocket去实现电脑浏览器与手机端的联结

**答题思路**：WebSocket 本身需要一个服务端做中转，「不借助服务器」通常指**不写自己的后端业务服务器**，可借助现成信令服务或点对点方案：

+ **公共/现成信令服务**：用第三方 WebSocket 服务（或本地起一个轻量 ws 服务）做**信令中转**，两端连同一个房间，互发消息实现联动。
+ **WebRTC 点对点**：真正的端到端只需 WebSocket 做一次**信令交换**（SDP、ICE candidate），协商完成后浏览器与手机走 P2P 直连传数据，媒体流不经服务器。
+ **同局域网**：手机扫码/输入局域网地址连到电脑本地起的 ws 服务，直接建连。

回答重点是说清「WebSocket 至少需要一个信令通道，真正的数据可以走 P2P」。

#### （7）大文件上传怎么实现的，如何利用原生API去实现大文件上传？

核心是**分片 + 并发 + 断点续传 + 秒传**：

+ **（1）分片**：用 `File.prototype.slice(start, end)` 把大文件切成多个 Blob 分片。
+ **（2）秒传/校验**：读取文件内容用 `FileReader`/`spark-md5` 计算哈希作为文件唯一标识，先问服务端是否已存在（存在则秒传）。
+ **（3）上传**：每片用 `fetch`/`XMLHttpRequest` 携带 hash + 分片序号上传，可并发控制。
+ **（4）断点续传**：上传前向服务端查询已上传的分片列表，跳过已传的，只传缺失分片。
+ **（5）合并**：全部上传完成后通知服务端按序合并分片为完整文件。

```javascript
const CHUNK = 5 * 1024 * 1024; // 5MB
const chunks = [];
for (let cur = 0; cur < file.size; cur += CHUNK) {
  chunks.push(file.slice(cur, cur + CHUNK));
}
```

#### （8）说一下Vue的双向绑定，以及你使用它的应用场景

**双向绑定** 指数据变化更新视图、视图（用户输入）变化也更新数据，Vue 中通过 `v-model` 实现。

+ **原理**：`v-model` 本质是**语法糖** = 绑定 `value`（数据→视图，靠响应式）+ 监听 `input` 事件（视图→数据）。底层依赖响应式系统（Vue2 `Object.defineProperty` / Vue3 `Proxy`）。
+ **场景**：表单输入（输入框、下拉、复选框）、搜索框、自定义组件的双向数据流。组件上使用时可通过 `modelValue` + `update:modelValue` 自定义（Vue3）。

#### （9）说一下Vue中的插槽使用，以及它的应用场景，默认插槽和具名插槽的区别与应用

**插槽（slot）** 让父组件把内容「插入」子组件的指定位置，实现内容分发与组件复用。

+ **默认插槽**：子组件用 `<slot>` 占位，父组件传入的内容填入唯一的默认插槽。适合内容不需要区分位置的情况。
+ **具名插槽**：子组件 `<slot name="header">`，父组件用 `<template #header>` 指定填入哪个插槽。适合布局型组件（如卡片的头/身/尾多个坑位）。
+ **作用域插槽**：子组件通过 `<slot :item="data">` 把内部数据暴露给父组件，父组件用 `#default="{ item }"` 接收。适合列表项、表格列等需要父级自定义渲染但用子级数据的场景。

#### （10）一道动态规划的题目（用dp、还有递归两种方式去实现即可）

**答题思路**：先定义状态与转移方程，再从递归（自顶向下 + 记忆化）过渡到 dp（自底向上）。以爬楼梯为例，`dp[i]` 表示到第 i 级台阶的走法数，转移方程 `dp[i] = dp[i-1] + dp[i-2]`。

```javascript
// 递归 + 记忆化
function climb(n, memo = {}) {
  if (n <= 2) return n;
  if (memo[n]) return memo[n];
  return (memo[n] = climb(n - 1, memo) + climb(n - 2, memo));
}
// 动态规划
function climbDP(n) {
  const dp = [1, 1];
  for (let i = 2; i <= n; i++) dp[i] = dp[i - 1] + dp[i - 2];
  return dp[n];
}
```

#### （11）你觉得你自己聪明嘛？你觉得自己有什么优势？如果实习过程中，遇到很大的技术难点，你愿意去花时间攻克这个东西嘛？责任心。
